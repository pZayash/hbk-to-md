## Why

Vault-страницы методов и свойств 1С содержат информацию о типах в виде нестуктурированной прозы (`Тип: [Строка](link).`). RAG-агенты получают страницу и вынуждены парсить типы из повествовательного текста без явного структурного сигнала. Добавление BSL-сигнатуры в начало каждой страницы создаёт компактный, семантически богатый summary-чанк с кликабельными ссылками на типы.

## What Changes

- Добавить новый этап постобработки `STAGE_INJECT_SIGNATURES` в pipeline `convert.py` — после `inject_breadcrumbs`, до `write_logs`.
- Для каждого content-файла vault парсить тело Markdown и извлекать: имена и типы параметров, тип возвращаемого значения, тип свойства.
- Вставлять сформированную BSL-сигнатуру сразу после H1 и breadcrumb, перед основным контентом страницы.
- Формат сигнатуры: inline backticks для имён, MD-ссылки для типов, `|` как разделитель составных типов, `→` для возвращаемого значения.
- Многовариантные методы (несколько синтаксических вариантов) — одна строка на вариант.

## Capabilities

### New Capabilities
- `bsl-type-signatures`: Генерация и вставка BSL-сигнатур с типами и ссылками в начало vault-страниц методов, свойств и конструкторов.

### Modified Capabilities
- `hbk-to-md-converter`: Добавляется новый этап пайплайна `STAGE_INJECT_SIGNATURES`; формат content-файлов расширяется блоком сигнатуры.

## Impact

- `convert.py`: новая функция-парсер Markdown-тела для извлечения типов, новая функция рендеринга сигнатурной строки, новый этап в `main()`.
- Vault content-файлы: изменяется структура (добавляется блок сигнатуры после H1/breadcrumb).
- Тесты: `tests/test_parse.py` нужно обновить под новый формат; возможно новый `tests/test_signatures.py`.
- `_meta.json`: не меняется.
- Index-файлы (`_index*.md`): не затрагиваются.
