## ADDED Requirements

### Requirement: BSL-сигнатура на страницах методов
Конвертер SHALL вставлять в начало каждой vault-страницы метода (включая многовариантные) строку BSL-сигнатуры, содержащую имя метода, параметры с типами и возвращаемый тип. Сигнатура SHALL использовать inline-backticks для имён и MD-ссылки для типов. Составные типы SHALL разделяться символом ` | `.

Формат одновариантного метода:
```
`МетодИмя`(`<Параметр1>`: [`Тип1`](link1), `<Параметр2>`: [`Тип2`](link2)) → [`ВозвращаемыйТип`](link)
```

Формат метода без возвращаемого значения:
```
`МетодИмя`(`<Параметр1>`: [`Тип1`](link1))
```

#### Scenario: Метод с параметрами и return-типом
- **WHEN** vault-страница метода содержит раздел `Синтаксис:`, параметры с блоками `Тип:` и раздел `Возвращаемое значение:` с `Тип:`
- **THEN** в файл вставлена строка вида `` `НайтиФайлы`(`<Путь>`: [`Строка`](lang__def_String.md), `<Маска>`: [`Строка`](lang__def_String.md), `<ИскатьВПодкаталогах>`: [`Булево`](lang__Булево_(Boolean).md)) → [`Массив`](Массив_(Array).md) ``
- **THEN** строка расположена после H1 и breadcrumb, перед первым контентным блоком

#### Scenario: Метод без возвращаемого значения (void)
- **WHEN** vault-страница метода не содержит раздел `Возвращаемое значение:`
- **THEN** сигнатура не содержит `→` и return-типа

#### Scenario: Параметр с отсутствующим блоком типа
- **WHEN** параметр не имеет строки `Тип:` в своём блоке
- **THEN** параметр включается в сигнатуру без аннотации типа: `` `<ПараметрБезТипа>` ``

### Requirement: BSL-сигнатура для многовариантных методов
Для методов с несколькими синтаксическими вариантами конвертер SHALL вставлять одну строку сигнатуры на каждый вариант. Строки SHALL разделяться одной пустой строкой.

#### Scenario: Два варианта синтаксиса
- **WHEN** vault-страница содержит два блока `Вариант синтаксиса:` с разными наборами параметров
- **THEN** вставлены две строки сигнатуры — по одной на каждый вариант
- **THEN** строки разделены пустой строкой

### Requirement: BSL-сигнатура на страницах свойств
Конвертер SHALL вставлять в начало каждой vault-страницы свойства строку BSL-сигнатуры, содержащую имя свойства и его тип (или составной тип). Возвращаемое значение не используется — вместо `→` используется `: `.

Формат:
```
`ИмяСвойства`: [`Тип`](link)
```

#### Scenario: Свойство с одним типом
- **WHEN** vault-страница свойства содержит строку `Тип: [Строка](link).` в разделе описания
- **THEN** вставлена строка `` `ИмяСвойства`: [`Строка`](link) ``

#### Scenario: Свойство с составным типом (nullable)
- **WHEN** строка `Тип:` содержит два типа, второй из которых `[Неопределено](...)`
- **THEN** вставлена строка `` `ЗащищенноеСоединение`: [`ЗащищенноеСоединениеOpenSSL`](link) | [`Неопределено`](link) ``

### Requirement: BSL-сигнатура на страницах конструкторов
Конвертер SHALL вставлять в начало vault-страниц конструкторов строку BSL-сигнатуры с параметрами. Конструктор не имеет return-типа.

#### Scenario: Конструктор с параметрами
- **WHEN** vault-страница содержит раздел `Синтаксис:` без раздела `Возвращаемое значение:`
- **WHEN** в пути страницы присутствует сегмент `ctors` (конструктор)
- **THEN** вставлена строка с именем и параметрами без `→`

### Requirement: Страницы без распознанного типа пропускаются
Если постобработчик не может определить тип страницы (метод/свойство/конструктор) или не находит ни одного блока `Тип:`, страница SHALL пропускаться без вставки сигнатуры и без ошибки.

#### Scenario: Страница без раздела Синтаксис и без Тип
- **WHEN** vault-страница не содержит ни `Синтаксис:`, ни `Тип:`
- **THEN** файл не модифицируется
- **THEN** счётчик `signatures_skipped` увеличивается на 1

### Requirement: Идемпотентность вставки сигнатуры
Повторный запуск конвертации SHALL заменять существующую сигнатуру, не создавая дублей.

#### Scenario: Повторный запуск с --clean
- **WHEN** конвертация запущена с `--clean` на ранее сконвертированный vault
- **THEN** каждая страница содержит ровно одну строку сигнатуры

#### Scenario: Повторный вызов STAGE_INJECT_SIGNATURES на уже обработанном vault
- **WHEN** `STAGE_INJECT_SIGNATURES` запущен на vault, где сигнатуры уже вставлены
- **THEN** файлы не изменяются (или содержат ту же сигнатуру — не дублированную)

### Requirement: Статистика сигнатур в _meta.json
`_meta.json` SHALL содержать дополнительные поля:
- `signatures_injected` — количество файлов, в которые вставлена сигнатура
- `signatures_skipped` — количество файлов, пропущенных без вставки

#### Scenario: Поля статистики записаны
- **WHEN** конвертация завершена
- **THEN** в `_meta.json.stats` есть `"signatures_injected": N, "signatures_skipped": M`
