# МенеджерПолнотекстовогоПоиска.УстановитьВерсиюПолнотекстовогоПоиска

МенеджерПолнотекстовогоПоиска (FullTextSearchManager)

УстановитьВерсиюПолнотекстовогоПоиска (SetFullTextSearchVersion)

Доступен, начиная с версии 8.3.22.

Синтаксис:

УстановитьВерсиюПолнотекстовогоПоиска(<ВерсияПолнотекстовогоПоиска>)

Параметры:

<ВерсияПолнотекстовогоПоиска> (обязательный)

Тип: [ВерсияПолнотекстовогоПоиска](ВерсияПолнотекстовогоПоиска_(FullTextSearchVersion).md).   
Новое устанавливаемое значение параметра <ВерсияПолнотекстовогоПоиска>.  
ВНИМАНИЕ! При использовании [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md), после полной индексации должно оставаться не менее 10% свободного пространства. В противном случае, индексация полнотекстового поиска выполнена не будет.

Описание:

Устанавливает (переключает) версию полнотекстового поиска. По умолчанию стоит версия [Версия1](ВерсияПолнотекстовогоПоиска.Версия1_(FullTextSearchVersion.Version1).md).   
Перед сменой версии требуется монопольный доступ к информационной базе. Если монопольный доступ не обеспечен, будет получена ошибка с сообщением о том, какие сеансы сейчас активны.

Доступность:

Сервер, толстый клиент, внешнее соединение.

Примечание:

ВНИМАНИЕ! При использовании [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md), после полной индексации должно оставаться не менее 10% свободного пространства на диске. В противном случае, индексация полнотекстового поиска выполнена не будет.  
  
При смене версии с [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md) на [Версия1](ВерсияПолнотекстовогоПоиска.Версия1_(FullTextSearchVersion.Version1).md) будут выполнены следующие действия:

- Все существующие (исполняющиеся) задания по индексации полнотекстового поиска будут прерваны;
- Все новые регламентные задания на индексацию от [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md) получат отказ;
- После первого отказа остановится индексация по расписанию для [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md);
- Все новые соединения с информационной базой будут использовать полнотекстовый поиск [Версия1](ВерсияПолнотекстовогоПоиска.Версия1_(FullTextSearchVersion.Version1).md);
- Индексы от [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md) остаются нетронутыми. Если индекс не нужен, перед переключением версии, пользователь может самостоятельно очистить индекс.

  
При смене версии с [Версия1](ВерсияПолнотекстовогоПоиска.Версия1_(FullTextSearchVersion.Version1).md) на [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md) будут выполнены следующие действия:


- Предварительная проверка доступности поискового сервера, если сервер недоступен, будет отображена ошибка и смены версии не произойдет. В сообщении будет указана причина недоступности сервера. Чаще всего причиной является отсутствие установленной Java;
- Все новые регламентные задания на индексацию будут направлены на новую версию 2;
- Все новые соединения с информационной базой получат полнотекстовый поиск версии [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md);
- Поисковый сервер [Версия2](ВерсияПолнотекстовогоПоиска.Версия2_(FullTextSearchVersion.Version2).md) всегда готов к работе и при первом обращении к нему он загрузит в память те индексы, что имеются в его распоряжении от предыдущего состояния. Так как это состояние, скорее всего, устаревшее, рекомендуется сделать полную переиндексацию;
- Индексы от "Версия1" остаются нетронутыми; если индекс вам не нужен, перед переключением версии, пользователь может самостоятельно очистить индекс.

Пример:

|  |
| --- |
| ПолнотекстовыйПоиск.УстановитьВерсиюПолнотекстовогоПоиска(ПолнотекстовыйПоиск.Версия2); |

Использование в версии:

Доступен, начиная с версии 8.3.22.
