# МенеджерКриптографии.НачатьУсовершенствованиеПодписи

**↑** [Главная](_index.md) › [Объекты](_index__Объекты.md) › [Общие объекты](Общие_объекты.md) › [Криптография](Криптография-2.md) › [МенеджерКриптографии](МенеджерКриптографии_(CryptoManager).md) › [Методы](МенеджерКриптографии__Методы.md)

МенеджерКриптографии (CryptoManager)

НачатьУсовершенствованиеПодписи (BeginEnhancingSignature)

Доступен, начиная с версии 8.3.20.

Вариант синтаксиса: Данные подписи в возвращаемом значении

Синтаксис:

НачатьУсовершенствованиеПодписи(<ОписаниеОповещения>, <ИсходныеДанные>, <ТипПодписи>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения усовершенствования со следующими параметрами:

- <ВыходныеДанные> - помещаются данные усовершенствованной подписи типа [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md) или [ФайловыйПоток](ФайловыйПоток_(FileStream).md).

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии_(CryptoSignatureType).md).   
Тип подписи криптографии.  
Значение по умолчанию: [CAdEST](ТипПодписиКриптографии.CAdEST_(CryptoSignatureType.CAdEST).md).

Описание варианта метода:

Данные подписи возвращаются методом как [Обещание](Обещание_(Promise).md) с результатом [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).

Вариант синтаксиса: Данные подписи сохраняются в файл

Синтаксис:

НачатьУсовершенствованиеПодписи(<ОписаниеОповещения>, <ИсходныеДанные>, <ВыходныеДанные>, <ТипПодписи>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения усовершенствования подписи со следующими параметрами:

- <ИмяФайла> - имя файла, в который осуществлен вывод.
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md) или [ФайловыйПоток](ФайловыйПоток_(FileStream).md).

<ВыходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Имя файла или поток, в который будут выгружены данные подписи.

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии_(CryptoSignatureType).md).   
Тип подписи криптографии.  
Значение по умолчанию: [CAdEST](ТипПодписиКриптографии.CAdEST_(CryptoSignatureType.CAdEST).md).

Описание варианта метода:

Данные подписи записываются в файл. Возвращает [Обещание](Обещание_(Promise).md) с результатом [Неопределено](lang__def_Undefined.md)

Описание:

Инициирует выполнение усовершенствования подписи в формате CAdES-BES.  
При создании усовершенствованной подписи выполняется обращение к серверам меток времени, адреса которых указаны в поле [АдресаСерверовМетокВремени](МенеджерКриптографии.АдресаСерверовМетокВремени_(CryptoManager.TimestampServersAddresses).md).  
Поддерживаемые форматы:

- [CAdEST](ТипПодписиКриптографии.CAdEST_(CryptoSignatureType.CAdEST).md),
- [CAdESC](ТипПодписиКриптографии.CAdESC_(CryptoSignatureType.CAdESC).md),
- [CAdESXLongType2](ТипПодписиКриптографии.CAdESXLongType2_(CryptoSignatureType.CAdESXLongType2).md),
- [CAdESAv3](ТипПодписиКриптографии.CAdESAv3_(CryptoSignatureType.CAdESAv3).md).

По завершении будет вызвана процедура, указанная в <ОписаниеОповещения>.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

Пример:

|  |
| --- |
| МенеджерКриптографии.НачатьУсовершенствованиеПодписи(         Оповещение, ПодписьCAdESBES, Результат, ТипПодписиКриптографии.CAdESXLongType2); |

Использование в версии:

Доступен, начиная с версии 8.3.20.

Описание изменено в версии 8.3.24.
