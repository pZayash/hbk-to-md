Конструкторы:

[По объектам используемых сертификатов](ЗащищенноеСоединениеOpenSSL.По_умолчанию.md)  

Описание:

Объект защищенного соединения OpenSSL.  
Используемый клиентский сертификат должен быть самоподписанным или проходить валидацию через сертификаты удостоверяющих центров, заданные в свойстве [СертификатыУдостоверяющихЦентров](ЗащищенноеСоединениеOpenSSL.СертификатыУдостоверяющихЦентров_(OpenSSLSecureConnection.CertificationAuthorityCertificates).md).

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

Пример:

|  |
| --- |
| ssl1 = Новый ЗащищенноеСоединениеOpenSSL(             Новый СертификатКлиентаОС,             Новый СертификатыУдостоверяющихЦентровОС,             РежимПроверкиОтзываСертификатовУдостоверяющихЦентров.Строгий);  ssl2 = Новый ЗащищенноеСоединениеOpenSSL(             Новый СертификатКлиентаОС(СпособВыбораСертификатаОС.Авто),             Новый СертификатыУдостоверяющихЦентровФайл("C:/certs/myca.pem"));  ssl3 = Новый ЗащищенноеСоединениеOpenSSL(             Новый СертификатКлиентаФайл("C:/certs/user.p12", "password" ),             Новый СертификатыУдостоверяющихЦентровФайл("C:/certs/myca.pem"));  ssl4 = Новый ЗащищенноеСоединениеOpenSSL(неопределено, неопределено); |
