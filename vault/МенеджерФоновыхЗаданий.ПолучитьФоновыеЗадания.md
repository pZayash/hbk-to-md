МенеджерФоновыхЗаданий (BackgroundJobsManager)

ПолучитьФоновыеЗадания (GetBackgroundJobs)

Доступен, начиная с версии 8.1.

Синтаксис:

ПолучитьФоновыеЗадания(<Отбор>)

Параметры:

<Отбор> (необязательный)

Тип: [Структура](Структура.md).   
Структура, задающая отбор. Значениями структуры могут быть: [УникальныйИдентификатор](ФоновоеЗадание.УникальныйИдентификатор.md), [Ключ](ФоновоеЗадание.Ключ.md), [Состояние](ФоновоеЗадание.Состояние.md), [Начало](ФоновоеЗадание.Начало.md), [Конец](ФоновоеЗадание.Конец.md), [Наименование](ФоновоеЗадание.Наименование.md), [ИмяМетода](ФоновоеЗадание.ИмяМетода.md), [РегламентноеЗадание](ФоновоеЗадание.РегламентноеЗадание.md).   
Если отбор не задан, получаются все задания.   
Если выполняется отбор по состояниям, то в качестве значения [Состояние](ФоновоеЗадание.Состояние.md) структуры может быть или перечисление [СостояниеФоновогоЗадания](СостояниеФоновогоЗадания.md), если идет выборка только по одному состоянию, или массив перечислений [СостояниеФоновогоЗадания](СостояниеФоновогоЗадания.md), если идет выборка по нескольким состояниям.   
Если выполняется отбор по регламентному заданию, то получаются все фоновые задания, которые были созданы по заданному регламентному заданию.   
Отбор по [Начало](ФоновоеЗадание.Начало.md) получает все фоновые задания, которые запущены после заданной даты.   
Отбор по [Конец](ФоновоеЗадание.Конец.md) получает все фоновые задания завершенные до заданной даты.

Возвращаемое значение:

Тип: [Массив](Массив.md).   

Описание:

Получает массив объектов [ФоновоеЗадание](ФоновоеЗадание.md) по заданному отбору. Получение заданий возможно только для администратора или пользователя, запустившего задания на выполнение.

Доступность:

Сервер, толстый клиент, внешнее соединение, мобильное приложение (сервер), мобильный автономный сервер.

Использование в версии:

Доступен, начиная с версии 8.1.
