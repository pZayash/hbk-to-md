Порядок (Order)

Добавить (Add)

Доступен, начиная с версии 8.0.

Синтаксис:

Добавить(<Описание>, <Имя>, <Представление>, <Направление>)

Параметры:

<Описание> (обязательный)

Тип: [Строка](lang__def_String.md).   
Описание добавляемого элемента порядка. Имя доступного поля и развернутый путь.  
Например, если есть доступное поле "Товар", то можно задать элемент порядка "Товар.СтранаПроисхождения.Наименование".

<Имя> (необязательный)

Тип: [Строка](lang__def_String.md).   
Имя элемента порядка. Если не задано, то создается имя по переданному описанию.

<Представление> (необязательный)

Тип: [Строка](lang__def_String.md).   
Пользовательское представление элемента порядка.

<Направление> (необязательный)

Тип: [НаправлениеСортировки](НаправлениеСортировки_(SortDirection).md).   
Направление сортировки добавляемого элемента порядка.  
Значение по умолчанию: [Возр](НаправлениеСортировки.Возр_(SortDirection.Asc).md).

Описание:

Добавляет элемент упорядочивания.  
Внимание! Метод не доступен для упорядочивания динамических списков (свойство Порядок объектов [СправочникСписок.](СправочникСписок.Имя_справочника_(CatalogList.Catalog_name).md)<[Имя справочника](СправочникСписок.Имя_справочника_(CatalogList.Catalog_name).md)>, [ДокументСписок.](ДокументСписок.Имя_документа_(DocumentList.Document_name).md)<[Имя документа](ДокументСписок.Имя_документа_(DocumentList.Document_name).md)> и т.д.).

Доступность:

Сервер, толстый клиент, внешнее соединение.

Пример:

|  |
| --- |
| ПостроительОтчета.Порядок.Добавить(ИмяПоля); |

Использование в версии:

Доступен, начиная с версии 8.0.
