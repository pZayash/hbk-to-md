# МенеджерКриптографии.УсовершенствоватьПодпись

МенеджерКриптографии (CryptoManager)

УсовершенствоватьПодпись (EnhanceSignature)

Доступен, начиная с версии 8.3.20.

Вариант синтаксиса: Данные подписи в возвращаемом значении

Синтаксис:

УсовершенствоватьПодпись(<ИсходныеДанные>, <ТипПодписи>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md) или [ФайловыйПоток](ФайловыйПоток_(FileStream).md).

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии_(CryptoSignatureType).md).   
Тип подписи криптографии.  
Методы, отвечающие за установку подписи, поддерживают только следующие типы подписей:

- CAdEST (CAdEST) ,
- CAdESC (CAdESC) ,
- CAdESXLongType2 (CAdESXLongType2) ,
- CAdESAv3 (CAdESAv3) ,
- CMS (CMS) .

  
Значение по умолчанию: [CAdEST](ТипПодписиКриптографии.CAdEST_(CryptoSignatureType.CAdEST).md).

Описание варианта метода:

Данные подписи возвращаются методом как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).

Вариант синтаксиса: Данные подписи сохраняются в файл

Синтаксис:

УсовершенствоватьПодпись(<ИсходныеДанные>, <ВыходныеДанные>, <ТипПодписи>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md) или [ФайловыйПоток](ФайловыйПоток_(FileStream).md).

<ВыходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Имя файла или поток, в который будут выгружены данные подписи.  
Если [Неопределено](lang__def_Undefined.md), то результат работы метода вернется в возвращаемом значении типа [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии_(CryptoSignatureType).md).   
Тип подписи криптографии.  
Методы, отвечающие за установку подписи, поддерживают только следующие типы подписей:

- CAdEST (CAdEST) ,
- CAdESC (CAdESC) ,
- CAdESXLongType2 (CAdESXLongType2) ,
- CAdESAv3 (CAdESAv3) ,
- CMS (CMS) .

  
Значение по умолчанию: [CAdEST](ТипПодписиКриптографии.CAdEST_(CryptoSignatureType.CAdEST).md).

Описание варианта метода:

Данные подписи записываются в файл. Возвращает [Неопределено](lang__def_Undefined.md).

Возвращаемое значение:

Тип: [Неопределено](lang__def_Undefined.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).   
Двоичные данные усовершенствованной подписи (если <ВыходныеДанные> [Неопределено](lang__def_Undefined.md)) иначе [Неопределено](lang__def_Undefined.md).

Описание:

Выполняет усовершенствование подписи в формате CAdES-BES.  
При создании усовершенствованной подписи выполняется обращение к серверам меток времени, адреса которых указаны в поле [АдресаСерверовМетокВремени](МенеджерКриптографии.АдресаСерверовМетокВремени_(CryptoManager.TimestampServersAddresses).md).  
Поддерживаемые форматы:

- [CAdEST](ТипПодписиКриптографии.CAdEST_(CryptoSignatureType.CAdEST).md),
- [CAdESC](ТипПодписиКриптографии.CAdESC_(CryptoSignatureType.CAdESC).md),
- [CAdESXLongType2](ТипПодписиКриптографии.CAdESXLongType2_(CryptoSignatureType.CAdESXLongType2).md),
- [CAdESAv3](ТипПодписиКриптографии.CAdESAv3_(CryptoSignatureType.CAdESAv3).md).

Доступность:

Тонкий клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

Пример:

|  |
| --- |
| МенеджерКриптографии.УсовершенствоватьПодпись(Данные, "result.sig", ТипПодписиКриптографии.CAdEST); |

Использование в версии:

Доступен, начиная с версии 8.3.20.

Описание изменено в версии 8.3.24.
