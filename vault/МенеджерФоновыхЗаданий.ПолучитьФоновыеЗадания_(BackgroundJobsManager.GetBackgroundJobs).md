МенеджерФоновыхЗаданий (BackgroundJobsManager)

ПолучитьФоновыеЗадания (GetBackgroundJobs)

Доступен, начиная с версии 8.1.

Синтаксис:

ПолучитьФоновыеЗадания(<Отбор>)

Параметры:

<Отбор> (необязательный)

Тип: [Структура](Структура_(Structure).md).   
Структура, задающая отбор. Значениями структуры могут быть: [УникальныйИдентификатор](ФоновоеЗадание.УникальныйИдентификатор_(BackgroundJob.UUID).md), [Ключ](ФоновоеЗадание.Ключ_(BackgroundJob.Key).md), [Состояние](ФоновоеЗадание.Состояние_(BackgroundJob.State).md), [Начало](ФоновоеЗадание.Начало_(BackgroundJob.Begin).md), [Конец](ФоновоеЗадание.Конец_(BackgroundJob.End).md), [Наименование](ФоновоеЗадание.Наименование_(BackgroundJob.Description).md), [ИмяМетода](ФоновоеЗадание.ИмяМетода_(BackgroundJob.MethodName).md), [РегламентноеЗадание](ФоновоеЗадание.РегламентноеЗадание_(BackgroundJob.ScheduledJob).md).   
Если отбор не задан, получаются все задания.   
Если выполняется отбор по состояниям, то в качестве значения [Состояние](ФоновоеЗадание.Состояние_(BackgroundJob.State).md) структуры может быть или перечисление [СостояниеФоновогоЗадания](СостояниеФоновогоЗадания_(BackgroundJobState).md), если идет выборка только по одному состоянию, или массив перечислений [СостояниеФоновогоЗадания](СостояниеФоновогоЗадания_(BackgroundJobState).md), если идет выборка по нескольким состояниям.   
Если выполняется отбор по регламентному заданию, то получаются все фоновые задания, которые были созданы по заданному регламентному заданию.   
Отбор по [Начало](ФоновоеЗадание.Начало_(BackgroundJob.Begin).md) получает все фоновые задания, которые запущены после заданной даты.   
Отбор по [Конец](ФоновоеЗадание.Конец_(BackgroundJob.End).md) получает все фоновые задания завершенные до заданной даты.

Возвращаемое значение:

Тип: [Массив](Массив_(Array).md).   

Описание:

Получает массив объектов [ФоновоеЗадание](ФоновоеЗадание_(BackgroundJob).md) по заданному отбору. Получение заданий возможно только для администратора или пользователя, запустившего задания на выполнение.

Доступность:

Сервер, толстый клиент, внешнее соединение, мобильное приложение (сервер), мобильный автономный сервер.

Использование в версии:

Доступен, начиная с версии 8.1.
