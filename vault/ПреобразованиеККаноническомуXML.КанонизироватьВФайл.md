ПреобразованиеККаноническомуXML (ConversionToCanonicalXML)

КанонизироватьВФайл (CanonicalizeToFile)

Доступен, начиная с версии 8.3.9.

Синтаксис:

КанонизироватьВФайл(<ДокументDOM>, <ИмяФайлаXML>, <ТипКаноническогоXML>, <ФильтрXPath>, <ПрефиксыДляВключения>)

Параметры:

<ДокументDOM> (обязательный)

Тип: [ДокументDOM](ДокументDOM.md).   
Документ DOM, который нужно записать в каноническом виде.

<ИмяФайлаXML> (обязательный)

Тип: Строка.   
Имя файла, в который будет записан текст XML.

<ТипКаноническогоXML> (необязательный)

Тип: [ТипКаноническогоXML](ТипКаноническогоXML.md).   
Тип алгоритма приведения к каноническому виду.  
Значение по умолчанию: [КаноническийXML1\_1](ТипКаноническогоXML.КаноническийXML1_1.md).

<ФильтрXPath> (необязательный)

Тип: [РезультатXPath](РезультатXPath.md), Строка.   
Если передана Строка, то перед преобразованием к каноническому виду будет выполнен запрос XPath из этой строки, и каноническая форма XML-документа будет содержать только результирующее подмножество документа.  
Если передан объект типа [РезультатXPath](РезультатXPath.md), то к каноническому виду будет преобразовано подмножество узлов из переданного параметра.  
Значение по умолчанию: Неопределено.

<ПрефиксыДляВключения> (необязательный)

Тип: Строка.   
Содержит список префиксов пространств имен для включения в формате: "Префикс1 Префикс2", и так далее.  
Префиксы для включения используются в алгоритме исключающего канонического XML (см. стандарт W3C Exclusive XML Canonicalization - https://www.w3.org/TR/xml-exc-c14n/).   
В остальных случаях параметр игнорируется.  
Значение по умолчанию: Неопределено.

Возвращаемое значение:

Результат сохраняется в файл.

Описание:

Преобразует документ DOM к каноническому виду.

Доступность:

Тонкий клиент, сервер, толстый клиент, внешнее соединение.

Использование в версии:

Доступен, начиная с версии 8.3.9.
