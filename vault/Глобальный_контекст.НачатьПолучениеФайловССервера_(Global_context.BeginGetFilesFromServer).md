# Глобальный контекст.НачатьПолучениеФайловССервера

Глобальный контекст (Global context)

НачатьПолучениеФайловССервера (BeginGetFilesFromServer)

Доступен, начиная с версии 8.3.15.

Вариант синтаксиса: С диалогом

Синтаксис:

НачатьПолучениеФайловССервера(<ПолучаемыеФайлы>, <ПараметрыДиалогаПолученияФайлов>, <ПараметрыПолученияАрхиваФайлов>)

Параметры:

<ПолучаемыеФайлы> (обязательный)

Тип: [Массив](Массив_(Array).md).   
Массив объектов типа [ОписаниеПередаваемогоФайла](ОписаниеПередаваемогоФайла_(TransferableFileDescription).md).  
Каждый объект описывает получаемый файл:

- [Хранение](ОписаниеПередаваемогоФайла.Хранение_(TransferableFileDescription.Location).md) - расположение данных во временном хранилище или в информационной базе. Значение не должно быть равно пустой строке. В противном случае будет сгенерировано сообщение об ошибке.
- [Имя](ОписаниеПередаваемогоФайла.Имя_(TransferableFileDescription.Name).md) - либо полный путь к файлу, либо относительный путь к файлу. Относительный путь будет объединен со значением параметра <БазовыйКаталог>.

<ПараметрыДиалогаПолученияФайлов> (необязательный)

Тип: [ПараметрыДиалогаПолученияФайлов](ПараметрыДиалогаПолученияФайлов_(GetFilesDialogParameters).md).   
Структура, содержащая параметры диалога, который будет показан пользователю.

<ПараметрыПолученияАрхиваФайлов> (необязательный)

Тип: [ПараметрыПолученияАрхиваФайлов](ПараметрыПолученияАрхиваФайлов_(GetFilesArchiveParameters).md).   
Параметры получения файлов в виде архива. Если параметр не задан, то получаемые файлы не будут упакованы в архив.

Описание варианта метода:

Начинает получение файлов, отображая диалог, и сохраняет их в локальную файловую систему пользователя.   
В веб-клиенте расширение работы с файлами не требуется.

Вариант синтаксиса: Без диалога

Синтаксис:

НачатьПолучениеФайловССервера(<ОписаниеОповещенияОЗавершении>, <ПолучаемыеФайлы>, <БазовыйКаталог>, <ПараметрыПолученияАрхиваФайлов>)

Параметры:

<ОписаниеОповещенияОЗавершении> (необязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения со следующими параметрами:

- <ПолученныеФайлы> – массив объектов типа [ОписаниеПереданногоФайла](ОписаниеПереданногоФайла_(TransferredFileDescription).md).
- <ДополнительныеПараметры> – значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<ПолучаемыеФайлы> (обязательный)

Тип: [Массив](Массив_(Array).md).   
Массив объектов типа [ОписаниеПередаваемогоФайла](ОписаниеПередаваемогоФайла_(TransferableFileDescription).md).  
Каждый объект описывает получаемый файл:

- [Хранение](ОписаниеПередаваемогоФайла.Хранение_(TransferableFileDescription.Location).md) - расположение данных во временном хранилище или в информационной базе. Значение не должно быть равно пустой строке. В противном случае будет сгенерировано сообщение об ошибке.
- [Имя](ОписаниеПередаваемогоФайла.Имя_(TransferableFileDescription.Name).md) - либо полный путь к файлу, либо относительный путь к файлу. Относительный путь будет объединен со значением параметра <БазовыйКаталог>.

<БазовыйКаталог> (необязательный)

Тип: [Строка](lang__def_String.md).   
Устанавливает базовый каталог, относительно которого определяется конечное расположение каждого из файлов.  
Если среди описаний получаемых файлов есть относительные пути или установлен режим получения архива [ПолучатьАрхивВсегда](РежимПолученияАрхиваФайлов.ПолучатьАрхивВсегда_(GetFilesArchiveMode.GetArchiveAlways).md), то параметр <БазовыйКаталог> должен быть задан обязательно.

<ПараметрыПолученияАрхиваФайлов> (необязательный)

Тип: [ПараметрыПолученияАрхиваФайлов](ПараметрыПолученияАрхиваФайлов_(GetFilesArchiveParameters).md).   
Параметры получения файлов в виде архива. Если параметр не задан, то получаемые файлы не будут упакованы в архив.

Описание варианта метода:

Начинает получение файлов и сохраняет их в локальную файловую систему пользователя.  
В веб-клиенте необходимо расширение работы с файлами.

Описание:

Начинает получение файлов.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, толстый клиент, мобильное приложение (клиент).
