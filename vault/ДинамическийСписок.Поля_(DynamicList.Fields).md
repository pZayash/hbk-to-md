ДинамическийСписок (DynamicList)

Поля (Fields)

Доступен, начиная с версии 8.3.19 (в режиме совместимости с версией 8.3.19 и последующими).

Использование:

Только чтение.

Описание:

Тип: [ПоляНабораДанныхСхемыКомпоновкиДанных](ПоляНабораДанныхСхемыКомпоновкиДанных_(DataCompositionSchemaDataSetFields).md).   
Содержит описания полей (набора данных) динамического списка.  
  
Свойства [Роль](ПолеНабораДанныхСхемыКомпоновкиДанных.Роль_(DataCompositionSchemaDataSetField.Role).md), [ВыражениеПредставления](ПолеНабораДанныхСхемыКомпоновкиДанных.ВыражениеПредставления_(DataCompositionSchemaDataSetField.PresentationExpression).md), [НаборДанныхПроверкиИерархии](ПолеНабораДанныхСхемыКомпоновкиДанных.НаборДанныхПроверкиИерархии_(DataCompositionSchemaDataSetField.HierarchyCheckDataSet).md) и [ПараметрНабораДанныхПроверкиИерархии](ПолеНабораДанныхСхемыКомпоновкиДанных.ПараметрНабораДанныхПроверкиИерархии_(DataCompositionSchemaDataSetField.HierarchyCheckDataSetParameter).md) не поддерживаются в динамическом списке.  
  
Кроме того, при установленной основной таблице:

- для многих стандартных полей (например, Ссылка, Наименование, ПометкаУдаления) не поддерживается установка значение свойства [Поле](ОграничениеИспользованияПоляСхемыКомпоновкиДанных.Поле_(DataCompositionSchemaFieldUseRestriction.Field).md) в [Истина](lang__def_BooleanTrue.md),
- для ключевых полей, стандартных полей сортировки и поля Родитель дополнительно не поддерживается установка значений свойств [Условие](ОграничениеИспользованияПоляСхемыКомпоновкиДанных.Условие_(DataCompositionSchemaFieldUseRestriction.Condition).md) и [Порядок](ОграничениеИспользованияПоляСхемыКомпоновкиДанных.Порядок_(DataCompositionSchemaFieldUseRestriction.Order).md) в [Истина](lang__def_BooleanTrue.md).

Доступность:

Сервер, мобильное приложение (сервер), мобильный автономный сервер.

Использование в версии:

Доступен, начиная с версии 8.3.19 (в режиме совместимости с версией 8.3.19 и последующими).
