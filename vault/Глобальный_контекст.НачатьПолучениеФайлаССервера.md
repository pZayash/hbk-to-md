Глобальный контекст (Global context)

НачатьПолучениеФайлаССервера (BeginGetFileFromServer)

Доступен, начиная с версии 8.3.15.

Вариант синтаксиса: С диалогом

Синтаксис:

НачатьПолучениеФайлаССервера(<Адрес>, <ИмяФайла>, <ПараметрыДиалогаПолученияФайлов>)

Параметры:

<Адрес> (обязательный)

Тип: Строка.   
Расположение данных во временном хранилище или в информационной базе.  
Параметр не должен быть пустой строкой. В противном случае будет сгенерировано сообщение об ошибке.

<ИмяФайла> (необязательный)

Тип: Строка.   
Имя файла, которое будет предложено пользователю для сохранения файла. Пользователь может изменить имя файла.  
Значение по умолчанию: Пустая строка.

<ПараметрыДиалогаПолученияФайлов> (необязательный)

Тип: [ПараметрыДиалогаПолученияФайлов](ПараметрыДиалогаПолученияФайлов.md).   
Структура, содержащая параметры диалога, который будет показан пользователю.

Описание варианта метода:

Начинает получение файла и сохраняет его в локальную файловую систему пользователя, предлагая пользователю выбрать месторасположение получаемого файла.  
В веб-клиенте расширение работы с файлами не требуется.  
В веб-клиенте в зависимости от настроек браузера предлагается сохранить файл с указанным именем файла.  
В тонком и толстом клиентах отображается диалог, позволяющий указать действие над полученным файлом – открыть его или сохранить в выбранном пользователем каталоге. Если параметр <ИмяФайла> не указан, то операция открытия файла недоступна. При открытии полученного файла выполняется проверка наличия приложения, ассоциированного с типом файла. Проверка выполняется по расширению файла. Если подходящее приложение не найдено, то отображается системный диалог выбора приложения для открытия файла. После выбора приложения файл открывается в нем. Отказ от выбора приложения приводит к отмене открытия файла.  
На мобильной платформе диалоговое окно выбора файла не отображается.

Вариант синтаксиса: Без диалога

Синтаксис:

НачатьПолучениеФайлаССервера(<ОписаниеОповещенияОЗавершении>, <Адрес>, <ПутьКФайлу>)

Параметры:

<ОписаниеОповещенияОЗавершении> (необязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения.md).   
Содержит описание процедуры, которая будет вызвана после завершения со следующими параметрами:

- <ПолученныйФайл> – объект типа [ОписаниеПереданногоФайла](ОписаниеПереданногоФайла.md).
- <ДополнительныеПараметры> – значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения.md).

<Адрес> (обязательный)

Тип: Строка.   
Расположение данных во временном хранилище или в информационной базе.  
Параметр не должен быть пустой строкой. В противном случае будет сгенерировано сообщение об ошибке.

<ПутьКФайлу> (обязательный)

Тип: Строка.   
Путь к файлу, в который будет сохранен файл.

Описание варианта метода:

Начинает получение файла и сохраняет его в локальную файловую систему пользователя по указанному пути. При этом пользователь не может изменить месторасположение получаемого файла.  
В веб-клиенте требуется расширение работы с файлами.

Описание:

Начинает получение файла.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, толстый клиент, мобильное приложение (клиент).

Использование в версии:

Доступен, начиная с версии 8.3.15.
