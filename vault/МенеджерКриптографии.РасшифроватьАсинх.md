МенеджерКриптографии (CryptoManager)

РасшифроватьАсинх (DecryptAsync)

Доступен, начиная с версии 8.3.18.

Вариант синтаксиса: Данные в двоичном виде

Синтаксис:

РасшифроватьАсинх(<ИсходныеДанные>, <Сертификат>, <ПарольДоступаКЗакрытомуКлючу>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: Строка, [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Зашифрованные данные.  
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md) или [ФайловыйПоток](ФайловыйПоток.md).

<Сертификат> (необязательный)

Тип: [СертификатКриптографии](СертификатКриптографии.md).   
Сертификат получателя зашифрованных данных, для которого выполняется расшифровка данных.

<ПарольДоступаКЗакрытомуКлючу> (необязательный)

Тип: Строка.   
Пароль для доступа к ключевому контейнеру получателя зашифрованных данных.

Описание варианта метода:

Передает расшифрованные данные в параметре типа [ДвоичныеДанные](ДвоичныеДанные.md).

Вариант синтаксиса: Сохранение данных в файл

Синтаксис:

РасшифроватьАсинх(<ИсходныеДанные>, <ВыходныеДанные>, <Сертификат>, <ПарольДоступаКЗакрытомуКлючу>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: Строка, [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Зашифрованные данные.  
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md) или [ФайловыйПоток](ФайловыйПоток.md).

<ВыходныеДанные> (обязательный)

Тип: Строка, [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Содержит имя выходного файла или поток, в который будут сохранены расшифрованные данные.

<Сертификат> (необязательный)

Тип: [СертификатКриптографии](СертификатКриптографии.md).   
Сертификат получателя зашифрованных данных, для которого выполняется расшифровка данных.

<ПарольДоступаКЗакрытомуКлючу> (необязательный)

Тип: Строка.   
Пароль для доступа к ключевому контейнеру получателя зашифрованных данных.

Описание варианта метода:

Помещает расшифрованные данные в файл.

Возвращаемое значение:

Тип: [Обещание](Обещание.md).   
Результат выполнения обещания - расшифрованные данные типа [ДвоичныеДанные](ДвоичныеДанные.md) или имя выходного файла с расшифрованными данными.

Описание:

Асинхронная функция. Расшифровывает зашифрованные данные.  
Чтобы метод успешно расшифровал данные, необходимо чтобы:

- В хранилище персональных сертификатов установлен сертификат, связанный с контейнером открытого ключа;
- Сертификат указан в качестве одного из получателей зашифрованных данных (перечислен в параметре <СписокПолучателей> метода [Зашифровать](МенеджерКриптографии.Зашифровать.md)).

Размер исходных зашифрованных данных ограничен 2 Гб (2147483647 байт).

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.
