КритерийОтбораМенеджер.<Имя критерия> (FilterCriterionManager.<Criterion name>)

ОбработкаПолученияФормы (FormGetProcessing)

Доступен, начиная с версии 8.3.1.

Синтаксис:

ОбработкаПолученияФормы(<ВидФормы>, <Параметры>, <ВыбраннаяФорма>, <ДополнительнаяИнформация>, <СтандартнаяОбработка>)

Параметры:

<ВидФормы>

Тип: [Строка](lang__def_String.md).   
Имя категории стандартной формы.

<Параметры>

Тип: [Структура](Структура_(Structure).md).   
Параметры формы.

<ВыбраннаяФорма>

Тип: [Строка](lang__def_String.md), [ОбъектМетаданных: Форма](ОбъектМетаданных_Форма_(MetadataObject_Form).md) .   
Содержит имя открываемой формы или объект метаданных Форма.  
Если значение параметра изменено, то будет осуществлен поиск формы в пределах объекта, для которого открывается форма.

<ДополнительнаяИнформация>

Тип: [Структура](Структура_(Structure).md).   
Дополнительная информация открытия формы.  
Структура, содержащая поле ПовторноеИспользование (Reuse) типа Булево, которое указывает, кешируется ли форма на стороне клиента.

<СтандартнаяОбработка>

Тип: [Булево](lang__Булево_(Boolean).md).   
В данный параметр передается признак выполнения стандартной (системной) обработки события. Если в теле процедуры-обработчика установить данному параметру значение [Ложь](lang__def_BooleanFalse.md), стандартная обработка события производиться не будет (выбираются установленные разработчиком форма и параметры).  
Значение по умолчанию: [Истина](lang__def_BooleanTrue.md).

Описание:

Обработчик события вызывается на сервере при получении стандартной формы клиентского приложения.   
Если требуется переопределить выбор открываемой формы, необходимо установить в параметре <ВыбраннаяФорма> другое имя формы или объект метаданных формы, которую требуется открыть, и в параметре <СтандартнаяОбработка> установить значение [Ложь](lang__def_BooleanFalse.md).

Доступность:

Сервер, толстый клиент, внешнее соединение.

Использование в версии:

Доступен, начиная с версии 8.3.1.
