ПланВидовРасчетаМенеджер.<Имя плана видов расчета> (ChartOfCalculationTypesManager.<Chart of calculation types name>)

УстановитьОбновлениеПредопределенныхДанных (SetPredefinedDataUpdate)

Доступен, начиная с версии 8.3.3.

Синтаксис:

УстановитьОбновлениеПредопределенныхДанных(<ОбновлениеПредопределенных>)

Параметры:

<ОбновлениеПредопределенных> (необязательный)

Тип: [ОбновлениеПредопределенныхДанных](ОбновлениеПредопределенныхДанных_(PredefinedDataUpdate).md).   
Режим обновления предопределенных объекта метаданных.  
Значение по умолчанию: Автоматически.

Описание:

Устанавливает режим обновления предопределенных элементов в данных. Для выполнения требуются права администратора информационной базы.  
Фактический режим обновления определяется в следующем порядке:

- Если для объекта метаданных в данных установлен режим обновления, отличный от [Авто](ОбновлениеПредопределенныхДанных.Авто_(PredefinedDataUpdate.Auto).md), то используется это значение.
- Иначе, если для объекта метаданных в конфигурации установлен режим обновления, отличный от [Авто](ОбновлениеПредопределенныхДанных.Авто_(PredefinedDataUpdate.Auto).md), то используется это значение.
- Иначе, если для информационной базы установлен режим обновления, отличный от [Авто](ОбновлениеПредопределенныхДанных.Авто_(PredefinedDataUpdate.Auto).md), то используется это значение.
- Иначе, если это периферийный узел РИБ, то предопределенные данные не будут обновлены. Если проверка выполняется для центрального узла РИБ, или для базы не являющейся РИБ, обновление предопределенных данных будет выполнено.

Метод доступен только из сеанса, в котором не используется ни один разделитель.

Доступность:

Сервер, толстый клиент, внешнее соединение.  
Вызов метода выполняет обращение к серверу.

Примечание:

В режиме совместимости конфигурации [Версия8\_3\_4](РежимСовместимости.Версия8_3_4_(CompatibilityMode.Version8_3_4).md) доступность метода не зависит от использования в сеансе разделителей.  
В режиме совместимости конфигурации [Версия8\_3\_3](РежимСовместимости.Версия8_3_3_(CompatibilityMode.Version8_3_3).md) необходимость обновлять определяется следующим образом:

- Обновление не будет производиться:
  - если в метаданных или в данных установлено [НеОбновлятьАвтоматически](ОбновлениеПредопределенныхДанных.НеОбновлятьАвтоматически_(PredefinedDataUpdate.DontAutoUpdate).md),
  - если в метаданных или в данных установлено [Авто](ОбновлениеПредопределенныхДанных.Авто_(PredefinedDataUpdate.Auto).md), и текущий узел является периферийным.
- В противном случае предопределенные данные будут обновлены и/или созданы при первом обращении к данным.

Использование в версии:

Доступен, начиная с версии 8.3.3.
