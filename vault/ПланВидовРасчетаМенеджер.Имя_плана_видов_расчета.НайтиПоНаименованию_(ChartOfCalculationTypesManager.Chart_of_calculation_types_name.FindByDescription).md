# ПланВидовРасчетаМенеджер.<Имя плана видов расчета>.НайтиПоНаименованию

ПланВидовРасчетаМенеджер.<Имя плана видов расчета> (ChartOfCalculationTypesManager.<Chart of calculation types name>)

НайтиПоНаименованию (FindByDescription)

Доступен, начиная с версии 8.0.

Синтаксис:

НайтиПоНаименованию(<Наименование>, <ТочноеСоответствие>)

Параметры:

<Наименование> (обязательный)

Тип: [Строка](lang__def_String.md).   
Наименование вида расчета для поиска.

<ТочноеСоответствие> (необязательный)

Тип: [Булево](lang__Булево_(Boolean).md).   
Определяет режим поиска по полному соответствию.  
Поиск будет успешным, если строка поиска: в случае значения параметра [Ложь](lang__def_BooleanFalse.md) - будет соответствовать левой части наименования; в случае значения параметра [Истина](lang__def_BooleanTrue.md) - будет полностью совпадать с наименованием (за исключением "хвостовых" пробелов в наименовании).  
Значение по умолчанию: [Ложь](lang__def_BooleanFalse.md).

Возвращаемое значение:

Тип: [ПланВидовРасчетаСсылка.](ПланВидовРасчетаСсылка.Имя_плана_видов_расчета_(ChartOfCalculationTypesRef.Chart_of_calculation_types_name).md)<[Имя плана видов расчета](ПланВидовРасчетаСсылка.Имя_плана_видов_расчета_(ChartOfCalculationTypesRef.Chart_of_calculation_types_name).md)>, [Неопределено](lang__def_Undefined.md).   
Ссылка на найденный вид расчета. Если объект с заданным наименованием не найден, возвращает пустую ссылку на вид расчета.  
Если наименование не задано (длина = 0) и поиск выполняется по полному соответствию, то будет возвращено [Неопределено](lang__def_Undefined.md).

Описание:

Осуществляет поиск вида расчета по наименованию.

Доступность:

Сервер, толстый клиент, внешнее соединение.

Примечание:

Если существует несколько узлов с указанным наименованием, то будет найден только один из них.

Использование в версии:

Доступен, начиная с версии 8.0.
