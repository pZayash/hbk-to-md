# Менеджер COM-соединений.MaxConnections

Менеджер COM-соединений (COM connector)

MaxConnections (MaxConnections)

Доступен, начиная с версии 8.1.

Использование:

Только чтение.

Описание:

Тип: [Число](lang__def_Number.md).   
Определяет максимальное число одновременно существующих объектов [Внешнее соединение](Внешнее_соединение_(External_connection).md), созданных через данный менеджер COM-соединений. Число одновременно существующих соединений включает также число соединений, находящихся в пуле внешних соединений. Значением по умолчанию 0 (число одновременно существующих объектов [Внешнее соединение](Внешнее_соединение_(External_connection).md) не ограничено).  
Устанавливать свойству не нулевое значение можно только в том случае, если сам объект был создан в Multithreaded Apartment (MTA), что обеспечивает реальное функционирование самого менеджера и созданных им объектов [Внешнее соединение](Внешнее_соединение_(External_connection).md) в многопотоковой (multithreaded) среде. Это условие соблюдается в частности, если COM-соединитель был создан в приложении ASP.NET.   
Если же объект был создан в Singlethreaded Apartment (STA), то реально все действия с ним и созданными им объектами [Внешнее соединение](Внешнее_соединение_(External_connection).md) будут выполняться в одном потоке. В этом случае, если метод [Connect](Менеджер_COM-соединений.Connect_(COM_connector.Connect).md) не сможет на первом проходе найти подходящее или создать новое [Внешнее соединение](Внешнее_соединение_(External_connection).md), то выполнение этого метода зациклится, так как освободить уже, возможно, ненужное другое [Внешнее соединение](Внешнее_соединение_(External_connection).md) будет нельзя по той причине, что операция освобождения объекта [Внешнее соединение](Внешнее_соединение_(External_connection).md) должна быть выполнена в том же потоке, в котором уже выполняется метод [Connect](Менеджер_COM-соединений.Connect_(COM_connector.Connect).md).   
Следует помнить, что создание COM-объектов из приложения 1C:Предприятие 8 выполняется из STA. Это же справедливо для большинства GUI-приложений Windows.

Доступность:

Интеграция.

Использование в версии:

Доступен, начиная с версии 8.1.
