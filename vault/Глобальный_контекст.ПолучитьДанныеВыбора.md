Глобальный контекст (Global context)

ПолучитьДанныеВыбора (GetChoiceData)

Доступен, начиная с версии 8.2.

Синтаксис:

ПолучитьДанныеВыбора(<ТипЗначения>, <Параметры>)

Параметры:

<ТипЗначения> (обязательный)

Тип: Тип.   
Объект метаданных, для которого выполняется формирование типа значений.  
Указывается в виде типа.  
Например: СправочникСсылка.Товары.

<Параметры> (обязательный)

Тип: [Структура](Структура.md).   
Содержит параметры для формирования списка значений.  
Структура, в которой обязательно должны содержаться следующие ключи:

- Отбор (Filter) - тип [Структура](Структура.md). Отбор, используемый для поиска данных. Значения элементов отбора могут иметь тип [Массив](Массив.md), [СписокЗначений](СписокЗначений.md), [ФиксированныйМассив](ФиксированныйМассив.md) или иной тип, не являющийся коллекцией; для типов [Массив](Массив.md), [СписокЗначений](СписокЗначений.md), [ФиксированныйМассив](ФиксированныйМассив.md) сравнение выполняется с видом сравнения [ВСписке](ВидСравнения.ВСписке.md), иначе вид сравнения [Равно](ВидСравнения.Равно.md);
- СтрокаПоиска (SearchString) - содержит строку, используемую при поиске данных. Для быстрого выбора содержит значение Неопределено;
- ВыборГруппИЭлементов (ChoiceFoldersAndItems) (необязательный) - тип [ИспользованиеГруппИЭлементов](ИспользованиеГруппИЭлементов.md). Указывает использование групп и элементов;
- СпособПоискаСтроки (StringSearchMode) (необязательный) - тип [СпособПоискаСтрокиПриВводеПоСтроке](СпособПоискаСтрокиПриВводеПоСтроке.md). Указывает способ поиска при вводе по строке;
- ПолнотекстовыйПоиск (FullTextSearch) (необязательный) - тип [ПолнотекстовыйПоискПриВводеПоСтроке](ПолнотекстовыйПоискПриВводеПоСтроке.md). Указывает необходимость использования полнотекстового поиска при вводе по строке;
- ОграничиватьСтрокуПоиска (LimitSearchString) (необязательный) - тип Булево. Указывает необходимость применения ограничений на искомую строку при вводе по строке.

Возвращаемое значение:

Тип: [СписокЗначений](СписокЗначений.md).   

Описание:

Формирует список значений данных выбора при вводе по строке, автоподборе текста и быстром выборе.  
Если требуется выполнить поиск более 32 слов или 1000 символов, то свойство [ОграничиватьСтрокуПоиска](СписокПолнотекстовогоПоиска.ОграничиватьСтрокуПоиска.md) требуется установить в значение Ложь.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.  
Вызов метода выполняет обращение к серверу.

Примечание:

При вызове на тонком и веб-клиентах в режиме низкой скорости соединения возвращаемые значения кэшируются.  
Клиентский кэш сбрасывается, если два раза подряд были запрошены данные с одинаковыми значениями параметров.  
  
В режиме совместимости конфигурации [Версия8\_3\_17](РежимСовместимости.Версия8_3_17.md) и ниже рекомендуется устанавливать свойство [ОграничиватьСтрокуПоиска](СписокПолнотекстовогоПоиска.ОграничиватьСтрокуПоиска.md) в значение Истина, чтобы при выполнении сложных запросов генерировалась ошибка.

Использование в версии:

Доступен, начиная с версии 8.2.

Описание изменено в версии 8.3.18.
