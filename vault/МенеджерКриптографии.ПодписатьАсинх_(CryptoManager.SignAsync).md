# МенеджерКриптографии.ПодписатьАсинх

МенеджерКриптографии (CryptoManager)

ПодписатьАсинх (SignAsync)

Доступен, начиная с версии 8.3.18.

Вариант синтаксиса: Данные подписи в возвращаемом значении

Синтаксис:

ПодписатьАсинх(<ИсходныеДанные>, <Сертификат>, <ТипПодписи>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные для подписывания.  
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md) или [ФайловыйПоток](ФайловыйПоток_(FileStream).md).

<Сертификат> (необязательный)

Тип: [СертификатКриптографии](СертификатКриптографии_(CryptoCertificate).md).   
Используемый сертификат криптографии.  
Если параметр не указан, то в хранилище персональных сертификатов должен быть ровно один сертификат. В противном случае генерируется исключение.

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии_(CryptoSignatureType).md).   
Тип подписи криптографии.  
Методы, отвечающие за установку подписи, поддерживают только следующие типы подписей:

- CMS (CMS) ,
- CAdESBES (CAdESBES) ,
- CAdEST (CAdEST) ,
- CAdESC (CAdESC) ,
- CAdESXLongType2 (CAdESXLongType2) ,
- CAdESAv3 (CAdESAv3) .

  
Значение по умолчанию: ТипПодписиКриптографии.CAdESBES.

Описание варианта метода:

Начинает создание подписи в формате CMS (базируется на PKCS#7) и возвращает ее как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md) или сохраняет ее в файл.   
Подпись создается алгоритмами, которые заданы свойствами [АлгоритмХеширования](МенеджерКриптографии.АлгоритмХеширования_(CryptoManager.HashAlgorithm).md) и [АлгоритмПодписи](МенеджерКриптографии.АлгоритмПодписи_(CryptoManager.SignAlgorithm).md).  
В данные подписи исходный файл не включается.   
В данные подписи могут быть включены сертификаты, в зависимости от свойства [ВключениеСертификатовВПодпись](МенеджерКриптографии.ВключениеСертификатовВПодпись_(CryptoManager.IncludeCertificatesInSignature).md).  
Если сертификат не указан, то для подписи используется единственный установленный персональный сертификат.  
В случае успешного завершения метода объект [Обещание](Обещание_(Promise).md) будет содержать [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md) подписи.

Вариант синтаксиса: Сохранение данных в файл

Синтаксис:

ПодписатьАсинх(<ИсходныеДанные>, <ВыходныеДанные>, <ТипПодписи>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные для подписывания.  
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md) или [ФайловыйПоток](ФайловыйПоток_(FileStream).md).

<ВыходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md).   
Имя файла, в который будут выгружены данные подписи.  
Если [Неопределено](lang__def_Undefined.md), то результат работы метода вернется в возвращаемом значении типа [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии_(CryptoSignatureType).md).   
Тип подписи криптографии.  
Методы, отвечающие за установку подписи, поддерживают только следующие типы подписей:

- CMS (CMS) ,
- CAdESBES (CAdESBES) ,
- CAdEST (CAdEST) ,
- CAdESC (CAdESC) ,
- CAdESXLongType2 (CAdESXLongType2) ,
- CAdESAv3 (CAdESAv3) .

  
Значение по умолчанию: [CAdESBES](ТипПодписиКриптографии.CAdESBES_(CryptoSignatureType.CAdESBES).md).

Описание варианта метода:

Данные подписи записываются в файл.

Возвращаемое значение:

Тип: [Обещание](Обещание_(Promise).md).   
Результат выполнения обещания - подписанные данные типа [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md) или имя файла, в который осуществлен вывод.  
Подпись создается алгоритмами, которые заданы свойствами [АлгоритмХеширования](МенеджерКриптографии.АлгоритмХеширования_(CryptoManager.HashAlgorithm).md) и [АлгоритмПодписи](МенеджерКриптографии.АлгоритмПодписи_(CryptoManager.SignAlgorithm).md).  
В данные подписи исходный файл не включается.   
В данные подписи могут быть включены сертификаты, в зависимости от свойства [ВключениеСертификатовВПодпись](МенеджерКриптографии.ВключениеСертификатовВПодпись_(CryptoManager.IncludeCertificatesInSignature).md).  
Если сертификат не указан, то для подписи используется единственный установленный персональный сертификат.  
По завершении будет вызвана процедура, указанная в параметре &ОписаниеОповещения.

Описание:

Асинхронная функция. Создает подпись в формате PKCS#7.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.
