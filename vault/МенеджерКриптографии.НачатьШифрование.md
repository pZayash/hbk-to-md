МенеджерКриптографии (CryptoManager)

НачатьШифрование (BeginEncrypting)

Доступен, начиная с версии 8.3.6.

Вариант синтаксиса: Данные в возвращаемом значении

Синтаксис:

НачатьШифрование(<ОписаниеОповещения>, <ИсходныеДанные>, <Получатели>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения.md).   
Содержит описание процедуры, которая будет вызвана после завершения шифрования данных со следующими параметрами:

- <ПодписанныеДанные> - зашифрованные данные типа [ДвоичныеДанные](ДвоичныеДанные.md).
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения.md).

<ИсходныеДанные> (обязательный)

Тип: Строка, [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Исходные данные для шифрования.   
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md) или [ФайловыйПоток](ФайловыйПоток.md).

<Получатели> (обязательный)

Тип: [СертификатКриптографии](СертификатКриптографии.md), [Массив](Массив.md).   
Параметр может содержать один сертификат или массив сертификатов.  
Только получатели из указанного списка (получатели, обладающие переданными сертификатами) смогут расшифровать данные, используя свои закрытые ключи.

Описание варианта метода:

Начинает шифрование данных для определенного круга получателей. Только получатели из указанного списка смогут расшифровать данные, используя свои закрытые ключи.  
Формат зашифрованных данных - CMS (базируется на PKCS#7).  
Используется алгоритм, заданный в свойстве [АлгоритмШифрования](МенеджерКриптографии.АлгоритмШифрования.md). После завершения шифрования будет вызвана процедура. имя которой указано в ОписаниеОповещения.  
В процедуре, указанной в параметре ОписаниеОповещения, должны присутствовать параметры:

- <ЗашифрованныеДанные> - зашифрованные данные типа [ДвоичныеДанные](ДвоичныеДанные.md).
- <ДополнительныеПараметры>.

Вариант синтаксиса: Сохранение данных в файл

Синтаксис:

НачатьШифрование(<ОписаниеОповещения>, <ИсходныеДанные>, <ВыходныеДанные>, <Получатели>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения.md).   
Содержит описание процедуры, которая будет вызвана после завершения шифрования данных со следующими параметрами:

- <ИмяФайла> - имя файла, в который осуществлен вывод.
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения.md).

<ИсходныеДанные> (обязательный)

Тип: Строка, [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Исходные данные для шифрования.   
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md) или [ФайловыйПоток](ФайловыйПоток.md).

<ВыходныеДанные> (обязательный)

Тип: Строка.   
Имя файла, в который будут выгружены зашифрованные данные.

<Получатели> (обязательный)

Тип: [СертификатКриптографии](СертификатКриптографии.md), [Массив](Массив.md).   
Параметр может содержать один сертификат или массив объектов сертификатов.   
Только получатели из указанного списка (получатели, обладающие переданными сертификатами) смогут расшифровать данные, используя свои закрытые ключи.

Описание варианта метода:

Зашифрованные данные записываются в файл.

Описание:

Начинает шифрование данных для определенного круга получателей. Только получатели из указанного списка смогут расшифровать данные, используя свои закрытые ключи.  
Формат зашифрованных данных - CMS (базируется на PKCS#7).  
Размер входных данных для шифрования ограничен 2 Гб (2147483647 байт).  
Используется алгоритм, заданный в свойстве [АлгоритмШифрования](МенеджерКриптографии.АлгоритмШифрования.md).  
После завершения шифрования, будет вызвана процедура. имя которой указано в [ОписаниеОповещения](ОписаниеОповещения.md).

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.
