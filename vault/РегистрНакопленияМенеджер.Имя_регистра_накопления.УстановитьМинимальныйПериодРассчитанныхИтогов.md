РегистрНакопленияМенеджер.<Имя регистра накопления> (AccumulationRegisterManager.<Accumulation register name>)

УстановитьМинимальныйПериодРассчитанныхИтогов (SetMinTotalsPeriod)

Доступен, начиная с версии 8.3.3.

Синтаксис:

УстановитьМинимальныйПериодРассчитанныхИтогов(<Период>)

Параметры:

<Период> (обязательный)

Тип: Дата.   
Период, начиная с которого будут рассчитаны итоги.   
Если указана дата 01.01.2000 то это значит, что будут рассчитаны итоги на начало дня 01.01.2000.

Описание:

Устанавливает период, начиная с которого рассчитаны итоги. Влияет только на производительность получения остатков по данному регистру. Периодичность расчета - месяц. Период устанавливается на начало месяца.

Доступность:

Сервер, толстый клиент, внешнее соединение, мобильное приложение (сервер), мобильный автономный сервер.

Примечание:

Имеет смысл только для регистров остатков.   
Период, на который рассчитаны итоги, влияет только на производительность получения остатков по данному регистру. Для получения итогов до этой даты будут использоваться движения.  
Если для конфигурации установлен режим совместимости [Версия8\_3\_2](РежимСовместимости.Версия8_3_2.md) и ниже, то привыполнении метода генерируется исключение.

Пример:

|  |
| --- |
| Регистры.Остатки.УстановитьМинимальныйПериодРассчитанныхИтогов(Дата); |

Использование в версии:

Доступен, начиная с версии 8.3.3.
