МенеджерСистемыВзаимодействия (CollaborationSystemManager)

НачатьРегистрациюИнформационнойБазы (BeginInfoBaseRegistration)

Доступен, начиная с версии 8.3.10.

Синтаксис:

НачатьРегистрациюИнформационнойБазы(<ОписаниеОповещения>, <ПараметрыРегистрации>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения работы метода со следующими параметрами:

- <РезультатРегистрации> - тип [РезультатРегистрацииИнформационнойБазыСистемыВзаимодействия](РезультатРегистрацииИнформационнойБазыСистемыВзаимодействия_(CollaborationSystemInfoBaseRegistrationResult).md). Результат регистрации информационной базы в сервере взаимодействия.
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<ПараметрыРегистрации> (обязательный)

Тип: [ПараметрыРегистрацииИнформационнойБазыСистемыВзаимодействия](ПараметрыРегистрацииИнформационнойБазыСистемыВзаимодействия_(CollaborationSystemInfoBaseRegistrationParameters).md).   
Параметры регистрации базы в системе взаимодействия.  
Параметры [АдресСервера](ПараметрыРегистрацииИнформационнойБазыСистемыВзаимодействия.АдресСервера_(CollaborationSystemInfoBaseRegistrationParameters.ServerAddress).md) и [АдресЭлектроннойПочты](ПараметрыРегистрацииИнформационнойБазыСистемыВзаимодействия.АдресЭлектроннойПочты_(CollaborationSystemInfoBaseRegistrationParameters.Email).md) являются обязательными.

Описание:

Метод предназначен для регистрации информационной базы на сервере взаимодействия.  
Для регистрации требуется выполнить два вызова метода:

- Во время первого вызова требуется установить параметры <ИмяИнформационнойБазы> и <АдресЭлектроннойПочты>:
  - Если используется сервер "1С:Диалог", на указанный адрес будет отправлено письмо, содержащее код активации.
  - Если используется иной сервер взаимодействия, письмо отправлено не будет.
- Если во время отправки запроса произошла ошибка, будет сгенерировано исключение.
- Во время второго вызова следует указать <ИмяИнформационнойБазы> и <АдресЭлектроннойПочты>, переданные ранее и полученный <КодАктивации>. Если во время регистрации произошла ошибка, информация о ней будет получена в параметре <ТекстСообщения> описания оповещения.

Требуется наличие у текущего пользователя информационной базы права "Регистрация системы взаимодействия".

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, толстый клиент.

Примечание:

Если в параметрах регистрации не указан [АдресЭлектроннойПочты](ПараметрыРегистрацииИнформационнойБазыСистемыВзаимодействия.АдресЭлектроннойПочты_(CollaborationSystemInfoBaseRegistrationParameters.Email).md) и в информационной базе нет объекта [ДанныеРегистрацииИнформационнойБазыСистемыВзаимодействия](ДанныеРегистрацииИнформационнойБазыСистемыВзаимодействия_(CollaborationSystemInfoBaseRegistrationData).md), при вызове данного метода будет сгенерировано исключение.
