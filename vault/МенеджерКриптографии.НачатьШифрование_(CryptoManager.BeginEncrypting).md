# МенеджерКриптографии.НачатьШифрование

МенеджерКриптографии (CryptoManager)

НачатьШифрование (BeginEncrypting)

Доступен, начиная с версии 8.3.6.

Вариант синтаксиса: Данные в возвращаемом значении

Синтаксис:

НачатьШифрование(<ОписаниеОповещения>, <ИсходныеДанные>, <Получатели>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения шифрования данных со следующими параметрами:

- <ПодписанныеДанные> - зашифрованные данные типа [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные для шифрования.   
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md) или [ФайловыйПоток](ФайловыйПоток_(FileStream).md).

<Получатели> (обязательный)

Тип: [СертификатКриптографии](СертификатКриптографии_(CryptoCertificate).md), [Массив](Массив_(Array).md).   
Параметр может содержать один сертификат или массив сертификатов.  
Только получатели из указанного списка (получатели, обладающие переданными сертификатами) смогут расшифровать данные, используя свои закрытые ключи.

Описание варианта метода:

Начинает шифрование данных для определенного круга получателей. Только получатели из указанного списка смогут расшифровать данные, используя свои закрытые ключи.  
Формат зашифрованных данных - CMS (базируется на PKCS#7).  
Используется алгоритм, заданный в свойстве [АлгоритмШифрования](МенеджерКриптографии.АлгоритмШифрования_(CryptoManager.EncryptAlgorithm).md). После завершения шифрования будет вызвана процедура. имя которой указано в [ОписаниеОповещения](html.md).  
В процедуре, указанной в параметре [ОписаниеОповещения](html.md), должны присутствовать параметры:

- <ЗашифрованныеДанные> - зашифрованные данные типа [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).
- <ДополнительныеПараметры>.

Вариант синтаксиса: Сохранение данных в файл

Синтаксис:

НачатьШифрование(<ОписаниеОповещения>, <ИсходныеДанные>, <ВыходныеДанные>, <Получатели>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения шифрования данных со следующими параметрами:

- <ИмяФайла> - имя файла, в который осуществлен вывод.
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные для шифрования.   
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md) или [ФайловыйПоток](ФайловыйПоток_(FileStream).md).

<ВыходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md).   
Имя файла, в который будут выгружены зашифрованные данные.

<Получатели> (обязательный)

Тип: [СертификатКриптографии](СертификатКриптографии_(CryptoCertificate).md), [Массив](Массив_(Array).md).   
Параметр может содержать один сертификат или массив объектов сертификатов.   
Только получатели из указанного списка (получатели, обладающие переданными сертификатами) смогут расшифровать данные, используя свои закрытые ключи.

Описание варианта метода:

Зашифрованные данные записываются в файл.

Описание:

Начинает шифрование данных для определенного круга получателей. Только получатели из указанного списка смогут расшифровать данные, используя свои закрытые ключи.  
Формат зашифрованных данных - CMS (базируется на PKCS#7).  
Размер входных данных для шифрования ограничен 2 Гб (2147483647 байт).  
Используется алгоритм, заданный в свойстве [АлгоритмШифрования](МенеджерКриптографии.АлгоритмШифрования_(CryptoManager.EncryptAlgorithm).md).  
После завершения шифрования, будет вызвана процедура. имя которой указано в [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.
