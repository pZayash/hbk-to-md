МенеджерКриптографии (CryptoManager)

С инициализацией модуля криптографии

Доступен, начиная с версии 8.2.

Синтаксис:

Новый МенеджерКриптографии(<ИмяМодуляКриптографии>, <ПутьМодуляКриптографии>, <ТипМодуляКриптографии>, <ИспользованиеИнтерактивногоРежима>)

Параметры:

<ИмяМодуляКриптографии> (обязательный)

Тип: [Строка](lang__def_String.md).   
Имя модуля криптографии. Написание без учета регистра.   
Используется для операционной системы MS Windows.

<ПутьМодуляКриптографии> (обязательный)

Тип: [Строка](lang__def_String.md).   
Путь к файлу-библиотеке, реализующей требуемый набор функций. В случае указания нескольких файлов-библиотек разделителем является символ двоеточия ":".  
В списке библиотек допускается наличие несуществующих файлов.  
Если задана пустая строка, то выполняется поиск файла-библиотеки в стандартных каталогах. Также в файле conf.cfg, в параметре CryptoAPILibraryLocation можно указать пути к файлу библиотеки или список путей, разделенных точками с запятой.  
Используется в Linux и macOS.

<ТипМодуляКриптографии> (обязательный)

Тип: [Число](lang__def_Number.md).   
Определяет тип модуля криптографии.

<ИспользованиеИнтерактивногоРежима> (необязательный)

Определяет режим работы модуля криптографии.

Описание:

Если модуль криптографии не найден, генерируется исключительная ситуация.  
На мобильной платформе игнорируются следующие параметры конструктора: <ИмяМодуляКриптографии>, <ПутьМодуляКриптографии>, <ТипМодуляКриптографии>.  
В веб-клиенте, если не установлено расширение работы с криптографией или если недоступно использование синхронных вызовов, при вызове конструктора генерируется исключение.

Использование в версии:

Доступен, начиная с версии 8.2.

Описание изменено в версии 8.3.24.
