Соединение с агентом сервера (IServerAgentConnection)

SetClusterRecyclingByMemory (SetClusterRecyclingByMemory)

Доступен, начиная с версии 8.2.

Синтаксис:

SetClusterRecyclingByMemory(<Кластер>, <ОбъемПамяти>, <Длительность>)

Параметры:

<Кластер> (обязательный)

Тип: [Кластер серверов](Кластер_серверов_(IClusterInfo).md).   
Кластер, для которого требуется изменить настройку автоматического перезапуска рабочих процессов. Должен быть получен методом [GetClusters](Соединение_с_агентом_сервера.GetClusters_(IServerAgentConnection.GetClusters).md).

<ОбъемПамяти> (обязательный)

Тип: [Число](lang__def_Number.md).   
Критический объем виртуальной памяти рабочего процесса в килобайтах.   
0 - критический объем памяти не задан и автоматический перезапуск рабочих процессов не производится.

<Длительность> (обязательный)

Тип: [Число](lang__def_Number.md).   
Максимально допустимое время превышения рабочим процессом критического объема виртуальной памяти, в секундах.   
0 - время не ограничено и автоматический перезапуск рабочих процессов не производится.

Описание:

Устанавливает автоматический перезапуск рабочих процессов по объему памяти.   
Если объем занятого рабочим процессом виртуального адресного пространства превышает критический объем памяти постоянно на протяжении максимально допустимого времени, то рабочий процесс будет автоматически перезапущен.

Доступность:

Интеграция.

Использование в версии:

Доступен, начиная с версии 8.2.
