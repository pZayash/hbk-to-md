АдминистрированиеРабочийСервер (AdministrationWorkServer)

ВременноДопустимыйОбъемПамятиПроцессов (TemporaryAllowedProcessesTotalMemory)

Доступен, начиная с версии 8.3.15.

Использование:

Чтение и запись.

Описание:

Тип: Число.   
Определяет размер временно допустимого объема памяти процессов кластера (рабочих процессов и менеджеров).  
Суммарный объем виртуального адресного пространства, занимаемого процессами этого кластера на этом рабочем сервере, превышение которого может привести к снижению производительности рабочего сервера.  
Допустимые значения от -1 до 9223372036854775807.  
Если значение 0, то временно допустимый объем памяти процессов кластера определяется автоматически как 80% объема оперативной памяти рабочего сервера.  
Если значение -1, то временно допустимый памяти процессов кластера не ограничен.  
При превышении значения данного параметра сервер считается непроизводительным, и на него перестают назначаться новые соединения.  
Если параметр [ПериодПревышенияВременноДопустимогоОбъемаПамятиПроцессов](АдминистрированиеРабочийСервер.ПериодПревышенияВременноДопустимогоОбъемаПамятиПроцессов.md) отличен от 0, то при превышении временно допустимого объема на время большее, чем указанной в значении периода, рабочие процессы кластера завершаются в порядке убывания объема памяти, занимаемой процессом, пока суммарный объем оставшихся процессов на данном рабочем сервере перестанет превышать значение [ВременноДопустимыйОбъемПамятиПроцессов](АдминистрированиеРабочийСервер.ВременноДопустимыйОбъемПамятиПроцессов.md).

Доступность:

Тонкий клиент, сервер, толстый клиент, интеграция.

Использование в версии:

Доступен, начиная с версии 8.3.15.
