МенеджерИсторииДанных (DataHistoryManager)

ВыбратьВерсии (SelectVersions)

Доступен, начиная с версии 8.3.11.

Синтаксис:

ВыбратьВерсии(<Отбор>, <Колонки>, <Порядок>, <МаксимальноеКоличество>)

Параметры:

<Отбор> (обязательный)

Тип: [Структура](Структура_(Structure).md).   
Структура, содержащая условия отбора версий. В качестве пары "Ключ" - "Значение" используются:

- Метаданные (Metadata) - значение типа [ОбъектМетаданных: Справочник](ОбъектМетаданных_Справочник_(MetadataObject_Catalog).md), [ОбъектМетаданных: Документ](ОбъектМетаданных_Документ_(MetadataObject_Document).md), [ОбъектМетаданных: РегистрСведений](ОбъектМетаданных_РегистрСведений_(MetadataObject_InformationRegister).md), [ОбъектМетаданных: БизнесПроцесс](ОбъектМетаданных_БизнесПроцесс_(MetadataObject_BusinessProcess).md), [ОбъектМетаданных: Задача](ОбъектМетаданных_Задача_(MetadataObject_Task).md), [ОбъектМетаданных: ПланСчетов](ОбъектМетаданных_ПланСчетов_(MetadataObject_ChartOfAccounts).md), [ОбъектМетаданных: ПланВидовХарактеристик](ОбъектМетаданных_ПланВидовХарактеристик_(MetadataObject_ChartOfCharacteristicTypes).md), [ОбъектМетаданных: ПланОбмена](ОбъектМетаданных_ПланОбмена_(MetadataObject_ExchangePlan).md), [ОбъектМетаданных: ПланВидовРасчета](ОбъектМетаданных_ПланВидовРасчета_(MetadataObject_ChartOfCalculationTypes).md), [ОбъектМетаданных: Константа](ОбъектМетаданных_Константа_(MetadataObject_Constant).md).
- Используется для отбора версий по объекту метаданных. Если для объекта метаданных настроены ограничения прав доступа на уровне записей (RLS) или не установлено право "ЧтениеИсторииОтсутствующихДанных", то выполняется чтение данных. Если данные отсутствуют, то версия исключается из выборки.
- Данные (Data) - значение типа [СправочникСсылка.](СправочникСсылка.Имя_справочника_(CatalogRef.Catalog_name).md)<[Имя справочника](СправочникСсылка.Имя_справочника_(CatalogRef.Catalog_name).md)>, бизнес-процесса](БизнесПроцессСсылка.Имя_бизнес-процесса_(BusinessProcessRef.Business_process_name).md)>, [ЗадачаСсылка.](ЗадачаСсылка.Имя_задачи_(TaskRef.Task_name).md)<[Имя задачи](ЗадачаСсылка.Имя_задачи_(TaskRef.Task_name).md)>, [ПланСчетовСсылка.](ПланСчетовСсылка.Имя_плана_счетов_(ChartOfAccountsRef.Chart_of_accounts_name).md)<[Имя плана [ПланОбменаСсылка.](ПланОбменаСсылка.Имя_плана_обмена_(ExchangePlanRef.Exchange_plan_name).md)<[Имя плана обмена](ПланОбменаСсылка.Имя_плана_обмена_(ExchangePlanRef.Exchange_plan_name).md)>, [КонстантаКлючЗначения.](КонстантаКлючЗначения.Имя_константы_(ConstantValueKey.Constant_name).md)<[Имя константы](КонстантаКлючЗначения.Имя_константы_(ConstantValueKey.Constant_name).md)>. Используется для отбора версий конкретного объекта данных или записи регистра сведений. Если для объекта метаданных настроено ограничение прав доступа на уровне записей (RLS) или нет права "ЧтениеИсторииОтсутствующихДанных", то выполняется чтение данных. Если данные отсутствуют, генерируется исключение.
- ПредставлениеДанных (DataPresentation) - значение типа [Строка](lang__def_String.md). Для поиска по вхождению переданного значения в представлении объекта.
- НомерВерсии (VersionNumber) - значение типа [Число](lang__def_Number.md). Для отбора по номеру версии.
- ДатаНачала (StartDate) - значение типа [Дата](lang__def_Date.md). Отбор версий с датой создания, большей или равной указанной дате.
- ДатаОкончания (EndDate) - значение типа [Дата](lang__def_Date.md). Отбор версий с датой создания, меньшей или равной указанной дате.
- Пользователь (User) - значение типа [УникальныйИдентификатор](УникальныйИдентификатор_(UUID).md) или массив таких объектов. Отбор версий по идентификатору пользователя, создавшего версию. Условия, созданные по значениям, переданным в массиве, объединяются по ИЛИ.
- ИмяПользователя (UserName) - значение типа [Строка](lang__def_String.md) или массив таких объектов. Отбор версий по имени пользователя, создавшего версию, проверка на вхождение. Имена должны соответствовать именам, возвращаемым методом глобального контекста [ИмяПользователя](Глобальный_контекст.ИмяПользователя_(Global_context.UserName).md) (UserName). Условия, созданные по значениям, переданным в массиве, объединяются по ИЛИ.
- ПолноеИмяПользователя (UserFullName) - значение типа [Строка](lang__def_String.md) или массив таких объектов. Отбор версий по имени пользователя, создавшего версию, проверка на вхождение. Имена должны соответствовать именам, возвращаемым методом [ПолноеИмяПользователя](Глобальный_контекст.ПолноеИмяПользователя_(Global_context.UserFullName).md). Условия, созданные по значениям, переданным в массиве, объединяются по ИЛИ.
- ВидИзмененияДанных (DataChangeType) - значение из перечисления [ВидИзмененияДанных](ВидИзмененияДанных_(DataChangeType).md) или массив [ВидИзмененияДанных](ВидИзмененияДанных_(DataChangeType).md). Отбор по виду изменения.
- Комментарий (Comment) - значение типа [Строка](lang__def_String.md). Отбор по комментарию, проверяется вхождение строки.
- Транзакция(Transaction) - значение типа [Строка](lang__def_String.md). Отбор по транзакции, проверка на равенство.
- Узел (Node) - [ПланОбменаСсылка.](ПланОбменаСсылка.Имя_плана_обмена_(ExchangePlanRef.Exchange_plan_name).md)<[Имя плана обмена](ПланОбменаСсылка.Имя_плана_обмена_(ExchangePlanRef.Exchange_plan_name).md)>. Отбор по узлу плана обмена.
- ЗначенияПолей (FieldValues) - значение типа [Массив](Массив_(Array).md). Отбор по значению поля. Массив содержит элементы типа [Структура](Структура_(Structure).md), описывающие условия, накладываемые на данные версии. Отбираются версии, которые удовлетворяют всем условиям списка (условия объединяются по И):
  - Поле (Field) - значение типа [Строка](lang__def_String.md). Обязательный. Предназначен для поиска версий, содержащих указанное поле. Если указана табличная часть, то выполняется поиск по всем дочерним полям.
  - ЗначениеПослеИзменения (ValueAfterChange) - значение типа Произвольный. Выполняется поиск версий с данными, содержащими указанный объект метаданных, и значением, равным переданному значению. Отбираются все версии, в которых значение установлено. ЗначениеПослеИзменения должно быть заполнено, если не заполнено ЗначениеДоИзменения.
  - ЗначениеДоИзменения (ValueBeforeChange) - значение типа Произвольный. Выполняется поиск версий с данными, содержащими указанный объект метаданных, и предыдущим значением, равным переданному значению. Отбираются все версии, в которых в предыдущей версии установлено значение, а не только версия, в которой изменено значение. ЗначениеДоИзменения должно быть заполнено, если не заполнено ЗначениеПослеИзменения.
- ИзменениеЗначенийПолей (FieldValuesChange) - значение типа [Массив](Массив_(Array).md). Отбор по факту изменения значения поля версии. Массив содержит строки с именем поля. Строка содержит имя объекта метаданных, например, если необходимо выполнить отбор по реквизиту "Цена" объекта "Заказ", указывается "Цена", если по реквизиту "Цена" табличной части "ПозицииЗаказа", то указывается "ПозицииЗаказа.Цена". Позволяет выполнить поиск по удаленному объекту метаданных, или полученному по обмену. Версия, полученная по обмену, может содержать объекты метаданных, которых нет в конфигурации текущей информационной базы, поэтому такие реквизиты можно выбрать только по полному имени. Если по строке не найден объект метаданных, условие игнорируется.

Обязательно необходимо указать значения "Метаданные" или "Данные". Остальные значения - не обязательные. Если заполнены оба поля ("Метаданные" и "Данные"), то объект метаданных объекта, переданного в параметре "Данные", должен совпадать с объектом метаданных, переданного в параметре "Метаданные".

<Колонки> (необязательный)

Тип: [Массив](Массив_(Array).md), [Строка](lang__def_String.md), [Неопределено](lang__def_Undefined.md).   
Массив имен колонок или строка, содержащая имена колонок, по которым производится выбор версий. В строке имена колонок должны быть разделены запятыми. Доступны колонки:

- Данные,
- ПредставлениеДанных,
- НомерВерсии,
- Дата,
- Пользователь,
- ИмяПользователя,
- ПолноеИмяПользователя,
- ВидИзмененияДанных,
- Комментарий,
- Транзакция,
- Узел.

Если [Неопределено](lang__def_Undefined.md) - в результате таблица будет содержать все колонки.  
Значение по умолчанию: [Неопределено](lang__def_Undefined.md).

<Порядок> (необязательный)

Тип: [Массив](Массив_(Array).md), [Строка](lang__def_String.md), [Неопределено](lang__def_Undefined.md).   
Строка или массив, содержащий имена колонок, по которым требуется сортировать выбранные элементы. После каждого имени колонки через пробел может быть указано направление сортировки. Направление определяется: "Убыв" ("Desc") - упорядочивать по убыванию; "Возр" ("Asc") - упорядочивать по возрастанию. По умолчанию сортировка производится по возрастанию.  
Если задано значение [Неопределено](lang__def_Undefined.md) используются следующие настройки порядка:

- При отборе по "Метаданные" - "Дата Убыв", "НомерВерсии Убыв",
- При отборе по "Данные" - "НомерВерсии Убыв".

  
Значение по умолчанию: [Неопределено](lang__def_Undefined.md).

<МаксимальноеКоличество> (необязательный)

Тип: [Неопределено](lang__def_Undefined.md), [Число](lang__def_Number.md).   
Максимальное количество получаемых версий.  
Если указано число, метод получает в указанном порядке версии с учетом установленного отбора в количестве, не превышающем значения параметра.   
[Неопределено](lang__def_Undefined.md) - получаются все версии.  
Значение по умолчанию: [Неопределено](lang__def_Undefined.md).

Описание:

Выполняет поиск версий истории данных в соответствии с отбором.

Доступность:

Сервер, толстый клиент, внешнее соединение.  
Вызов метода выполняет обращение к серверу.

|  |
| --- |
