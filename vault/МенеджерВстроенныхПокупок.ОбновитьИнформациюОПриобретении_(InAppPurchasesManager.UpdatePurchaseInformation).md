МенеджерВстроенныхПокупок (InAppPurchasesManager)

ОбновитьИнформациюОПриобретении (UpdatePurchaseInformation)

Доступен, начиная с версии 8.3.8.

Синтаксис:

ОбновитьИнформациюОПриобретении(<ИдентификаторыПокупок>, <ТипПокупки>, <АвтоПодтверждение>)

Параметры:

<ИдентификаторыПокупок> (обязательный)

Тип: [Массив](Массив_(Array).md).   
Массив идентификаторов покупок, информация о которых обновляется.

<ТипПокупки> (необязательный)

Тип: [ТипВстроеннойПокупки](ТипВстроеннойПокупки_(InAppPurchaseType).md), [Неопределено](lang__def_Undefined.md).   
Тип покупок, идентификаторы которых указаны в параметре <ИдентификаторыПокупок>.  
Данный параметр предназначен для оптимизации запросов в сервис покупок на Android.  
На других устройствах параметр игнорируется  
  
Если параметр указан, то обновлены будут только покупки указанного типа. Идентификаторы покупок с другим типом будут игнорироваться. Если параметр не использовать, то будет выполнено два запроса в сервис покупок, независимо от списка покупок. Указывайте параметр, когда в вашем списке покупок только подписки или наоборот только контент для продажи.  
Значение по умолчанию: [Неопределено](lang__def_Undefined.md).

<АвтоПодтверждение> (необязательный)

Тип: [Булево](lang__Булево_(Boolean).md).   
Если параметр установлен в значение [Истина](lang__def_BooleanTrue.md), то будет выполнена локальная проверка квитанции и если проверка успешна, то покупка будет подтверждена автоматически.   
Параметр используется на устройствах под управлением ОС Android. На iOS и Windows Phone игнорируется.   
Проверка квитанции требует наличия ключа проверки встроенных покупок в ресурсах приложения, который указывается в сборщике мобильных приложений в настройках приложения под Android. Если ключ не найден, то генерируется исключение "Ключ проверки квитанции не найден в ресурсах приложения".   
Значение по умолчанию зависит от режима совместимости:

- Версия
- До версии 8.3.18 - [Истина](lang__def_BooleanTrue.md).
- Версия 8.3.18 и последующие – [Ложь](lang__def_BooleanFalse.md).

  
Значение по умолчанию: Ложь.

Описание:

Получает список приобретенных покупок из сервиса-магазина. Вызов данного метода обновляет данные о приобретенных покупках, которые сохраняются операционной системой. Метод позволяет решить следующие задачи:

- восстановить информацию о приобретенных постоянных покупках и действующих подписках после того, как мобильное приложение было полностью удалено, а затем заново установлено;
- актуализировать информацию о приобретенных постоянных покупках и подписках, если пользователь пользуется приложением на нескольких своих устройствах;
- обновить и записать в локальное хранилище информацию о запрошенных покупках (описание, цена и др. поля), эти данные потом можно получить, с помощью метода [ПолучитьСписок](МенеджерВстроенныхПокупок.ПолучитьСписок_(InAppPurchasesManager.GetList).md).

Для работы метода обязательно активное подключение к сети Интернет. В случае его отсутствия будет сгенерировано исключение.  
Следует помнить, что обращение к сервису-магазину может занимать значительный период времени.  
Данный метод поддерживается не на всех устройствах. Для проверки совместимости устройства перед вызовом данного метода следует использовать метод [ПоддерживаютсяПокупки](МенеджерВстроенныхПокупок.ПоддерживаютсяПокупки_(InAppPurchasesManager.PurchasesSupported).md).

Доступность:

Мобильный клиент, мобильное приложение (клиент).
