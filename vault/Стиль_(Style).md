# Стиль

Доступен, начиная с версии 8.0.

Конструкторы:

[Конструктор по умолчанию](Стиль.По_умолчанию.md)  

Описание:

Представляет собой стиль, определенный в конфигурации. Может быть ссылкой на конкретный стиль конфигурации или значением, созданным конструктором стиля без параметров (стиль по умолчанию).  
Редактор стиля позволяет модифицировать для обычного приложения все предопределенные элементы стиля, а для приложения только следующие предопределенные цвета стиля:

- [ЦветАктивности](ЦветаСтиля.ЦветАктивности_(StyleColors.ActivityColor).md),
- [ЦветАкцента](ЦветаСтиля.ЦветАкцента_(StyleColors.AccentColor).md),
- [ЦветНавигации](ЦветаСтиля.ЦветНавигации_(StyleColors.NavigationColor).md),
- [ЦветДополнительнойНавигации](ЦветаСтиля.ЦветДополнительнойНавигации_(StyleColors.AuxiliaryNavigationColor).md),
- [ЦветВажного](ЦветаСтиля.ЦветВажного_(StyleColors.ImportantColor).md).

Также можно редактировать значения для собственных элементов стиля вида [Цвет](Цвет_(Color).md). Вне зависимости от возможности редактирования, существующие предопределенные элементы стиля разыменовываются в реальные цвета, используемые платформой. Для оформления элементов формы рекомендуется использовать именно цвета стиля, а не конкретные значения цвета. Другими словами, если для фона какого-либо элемента требуется использовать тот же цвет, что и цвет формы, то свойству, отвечающему за цвет фона элемента, следует установить значение стиля [ЦветФонаФормы](ЦветаСтиля.ЦветФонаФормы_(StyleColors.FormBackColor).md) (значение из системного набора значений [ЦветаСтиля](ЦветаСтиля_(StyleColors).md)), а не конкретное значение цвета. В этом случае, при изменении цвета фона формы, цвет фона выбранного элемента также изменится.  
Если необходимо отключить используемый стиль и установить стиль оформления платформы по умолчанию, следует использовать конструкцию вида:   

|  |
| --- |
| ГлавныйСтиль = Новый Стиль(); |

Доступность:

Тонкий клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение.
