ЗапускПриложенияМобильногоУстройства (MobileDeviceApplicationRun)

ПрикрепляемыеДанные (ClipData)

Доступен, начиная с версии 8.3.23.

Использование:

Чтение и запись.

Описание:

Тип: [ПрикрепляемыеДанныеЗапускаПриложенияМобильногоУстройства](ПрикрепляемыеДанныеЗапускаПриложенияМобильногоУстройства_(MobileDeviceApplicationRunClipData).md), [Неопределено](lang__def_Undefined.md).   
Содержит прикрепляемые данные, связанные с этим запуском приложения мобильного устройства. Прикрепляемые данные в запуске не используются для уточнения поиска мобильного приложения.   
Семантически похоже на свойство  
[ДополнительныеДанные](ЗапускПриложенияМобильногоУстройства.ДополнительныеДанные_(MobileDeviceApplicationRun.AdditionalData).md), используемое для передачи дополнительных данных с помощью [ЗапускПриложенияМобильногоУстройства](ЗапускПриложенияМобильногоУстройства_(MobileDeviceApplicationRun).md). Основная особенность использования этого свойства в сравнении со свойством [ДополнительныеДанные](ЗапускПриложенияМобильногоУстройства.ДополнительныеДанные_(MobileDeviceApplicationRun.AdditionalData).md) заключается в том, что системные флаги Intent.FLAG\_GRANT\_READ\_URI\_PERMISSION и Intent.FLAG\_GRANT\_WRITE\_URI\_PERMISSION, которые всегда устанавливаются объекту [ЗапускПриложенияМобильногоУстройства](ЗапускПриложенияМобильногоУстройства_(MobileDeviceApplicationRun).md), будут работать с любыми элементами со ссылками на ресурсы, включенными в прикрепляемые данные.   
Таким образом, при запуске мобильного приложения можно передать несколько ссылок на ресурсы со схемой "content:", для которых получатель может не иметь глобального разрешения на доступ к поставщику контента. Если данное свойство содержит элементы, которые сами являются объектами [ЗапускПриложенияМобильногоУстройства](ЗапускПриложенияМобильногоУстройства_(MobileDeviceApplicationRun).md), любые установленные флаги в этих запусках будут проигнорированы.   
Учитываются только флаги основного объекта [ЗапускПриложенияМобильногоУстройства](ЗапускПриложенияМобильногоУстройства_(MobileDeviceApplicationRun).md)(верхнего уровня) и они будут применены ко всем элементам со ссылками на ресурсы или объектам [ЗапускПриложенияМобильногоУстройства](ЗапускПриложенияМобильногоУстройства_(MobileDeviceApplicationRun).md) в ПрикрепляемыхДанных (или под элементами ПрикрепляемыхДанных).   
Тип, метка и значок MIME в свойстве ПрикрепляемыеДанные напрямую не используются объектом [ЗапускПриложенияМобильногоУстройства](ЗапускПриложенияМобильногоУстройства_(MobileDeviceApplicationRun).md). Приложения обычно должны учитывать MIME-тип самого объекта [ЗапускПриложенияМобильногоУстройства](ЗапускПриложенияМобильногоУстройства_(MobileDeviceApplicationRun).md), а не то, что находится в прикрепляемых данных.  
Обычно для передачи объекта [ЗапускПриложенияМобильногоУстройства](ЗапускПриложенияМобильногоУстройства_(MobileDeviceApplicationRun).md) следует создать объект [ПрикрепляемыеДанныеЗапускаПриложенияМобильногоУстройства](ПрикрепляемыеДанныеЗапускаПриложенияМобильногоУстройства_(MobileDeviceApplicationRunClipData).md) с типом MIME "\*/\*".  
Аналог методов setClipData и getClipData у класса Intent в операционной системе Android.

Доступность:

Мобильный клиент, мобильное приложение (клиент).

Примечание:

Значение по умолчанию - [Неопределено](lang__def_Undefined.md).

Использование в версии:

Доступен, начиная с версии 8.3.23.
