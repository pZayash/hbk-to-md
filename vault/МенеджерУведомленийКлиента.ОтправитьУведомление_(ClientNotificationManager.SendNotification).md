МенеджерУведомленийКлиента (ClientNotificationManager)

ОтправитьУведомление (SendNotification)

Доступен, начиная с версии 8.3.26.

Синтаксис:

ОтправитьУведомление(<Ключ>, <Данные>, <Адресаты>)

Параметры:

<Ключ> (необязательный)

Тип: [Строка](lang__def_String.md).   
Ключ уведомления.

<Данные> (обязательный)

Тип: Произвольный.   
Данные уведомления.

<Адресаты> (необязательный)

Тип: [Массив](Массив_(Array).md).   
Массив элементов типа [Число](lang__def_Number.md).  
Массив номеров сеанса, полученных с помощью свойства [НомерСеанса](СеансИнформационнойБазы.НомерСеанса_(InfoBaseSession.SessionNumber).md).  
Если параметр указан, то уведомление будет отправлено только определенным сеансам.  
Если не указан, уведомление будет отправлено всем сеансам.

Описание:

Отправляет уведомление клиентским сеансам.  
У клиентского сеанс может не быть соединения. Для гарантированной доставки уведомление сохраняется в данных сервиса уведомлений.

Доступность:

Сервер, толстый клиент, внешнее соединение.

Примечание:

Вызов метода выполняется до тех пор, пока не выполнено одно из следующих действий:

- уведомление успешно отправлено сеансу,
- уведомление записано на диск (в данные сервиса в кластере).

Использование в версии:

Доступен, начиная с версии 8.3.26.

Описание изменено в версии 8.3.27.
