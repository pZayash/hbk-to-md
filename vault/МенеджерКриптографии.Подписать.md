МенеджерКриптографии (CryptoManager)

Подписать (Sign)

Доступен, начиная с версии 8.2.

Вариант синтаксиса: Данные подписи сохраняются в файл

Синтаксис:

Подписать(<ИсходныеДанные>, <ВыходныеДанные>, <Сертификат>, <ТипПодписи>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: Строка, [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Исходные данные для подписывания.  
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные.md).

<ВыходныеДанные> (обязательный)

Тип: Строка, [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Имя файла или поток, в который будут выгружены данные подписи.  
Если Неопределено, то результат работы метода вернется в возвращаемом значении типа [ДвоичныеДанные](ДвоичныеДанные.md).

<Сертификат> (необязательный)

Тип: [СертификатКриптографии](СертификатКриптографии.md).   
Используемый сертификат криптографии.  
Если параметр не указан, то в хранилище персональных сертификатов должен быть ровно один сертификат. В противном случае генерируется исключительная ситуация.

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии.md).   
Тип подписи криптографии.  
Методы, отвечающие за установку подписи, поддерживают только следующие типы подписей:

- CMS (CMS) ,
- CAdESBES (CAdESBES) ,
- CAdEST (CAdEST) ,
- CAdESC (CAdESC) ,
- CAdESXLongType2 (CAdESXLongType2) ,
- CAdESAv3 (CAdESAv3) .

  
Значение по умолчанию: [CAdEST](ТипПодписиКриптографии.CAdEST.md).

Описание варианта метода:

Данные подписи записываются в файл. Возвращает Неопределено.

Возвращаемое значение:

Тип: Неопределено, [ДвоичныеДанные](ДвоичныеДанные.md).   
При сохранении подписи в файл возвращает Неопределено.

Описание:

Создает подпись в формате CMS (базируется на PKCS#7) и возвращает ее как [ДвоичныеДанные](ДвоичныеДанные.md) или сохраняет ее в файл.   
Подпись создается алгоритмами, которые заданы свойствами [АлгоритмХеширования](МенеджерКриптографии.АлгоритмХеширования.md) и [АлгоритмПодписи](МенеджерКриптографии.АлгоритмПодписи.md).  
В данные подписи исходный файл не включается.   
В данные подписи могут быть включены сертификаты, в зависимости от свойства [ВключениеСертификатовВПодпись](МенеджерКриптографии.ВключениеСертификатовВПодпись.md).  
Если сертификат не указан, то для подписи используется единственный установленный персональный сертификат.  
  
Для работы метода на веб-клиенте необходимо предварительно подключить расширение работы с криптографией. В веб-клиенте метод поддерживается только в браузере Internet Explorer, если в конфигурации используется режим синхронных вызовов расширений платформы и внешних компонент.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

Примечание:

Ошибочной является ситуация, когда персональные сертификаты не установлены.  
Ошибочной является ситуация, когда используемый для подписи сертификат не связан с контейнером ключей.
