СериализаторXDTO (XDTOSerializer)

ЗаписатьXML (WriteXML)

Доступен, начиная с версии 8.1.

Вариант синтаксиса: Записать без имени

Синтаксис:

ЗаписатьXML(<ЗаписьXML>, <Значение>, <НазначениеТипа>, <Форма>)

Параметры:

<ЗаписьXML> (обязательный)

Тип: [ЗаписьУзловDOM](ЗаписьУзловDOM.md), [ЗаписьFastInfoset](ЗаписьFastInfoset.md), [ЗаписьXML](ЗаписьXML.md).   
Объект, через который осуществляется запись XML.

<Значение> (обязательный)

Записываемое в поток XML значение. Тип параметра определяется совокупностью типов, для которых определена XML-сериализация.  
При записи объекта метаданных выгружаются все записи объекта метаданных, доступные для пользователя.

<НазначениеТипа> (необязательный)

Тип: [НазначениеТипаXML](НазначениеТипаXML.md).   
Определяет необходимость назначения типа элементу XML.  
Значение по умолчанию: [Неявное](НазначениеТипаXML.Неявное.md).

<Форма> (необязательный)

Тип: [ФормаXML](ФормаXML.md).   
Форма записи элемента данных в XML.

Описание варианта метода:

Записывает значения в формате XML без указания имени элемента. В качестве имени элемента будет использован тип значения.

Вариант синтаксиса: Записать с полным именем

Синтаксис:

ЗаписатьXML(<ЗаписьXML>, <Значение>, <ПолноеИмя>, <НазначениеТипа>, <Форма>)

Параметры:

<ЗаписьXML> (обязательный)

Тип: [ЗаписьУзловDOM](ЗаписьУзловDOM.md), [ЗаписьFastInfoset](ЗаписьFastInfoset.md), [ЗаписьXML](ЗаписьXML.md).   
Объект, через который осуществляется запись XML.

<Значение> (обязательный)

Записываемое в поток XML значение. Тип параметра определяется совокупностью типов, для которых определена XML-сериализация.  
При записи объекта метаданных выгружаются все записи объекта метаданных, доступные для пользователя.

<ПолноеИмя> (обязательный)

Тип: Строка.   
Полное имя элемента XML, в который будет записано значение.

<НазначениеТипа> (необязательный)

Тип: [НазначениеТипаXML](НазначениеТипаXML.md).   
Определяет необходимость назначения типа элементу XML.  
Значение по умолчанию: [Неявное](НазначениеТипаXML.Неявное.md).

<Форма> (необязательный)

Тип: [ФормаXML](ФормаXML.md).   
Форма записи элемента данных в XML.

Описание варианта метода:

Записывает значения в формате XML с указанием полного имени элемента.

Вариант синтаксиса: Записать с локальным именем и пространством имен

Синтаксис:

ЗаписатьXML(<ЗаписьXML>, <Значение>, <ЛокальноеИмя>, <URIПространстваИмен>, <НазначениеТипа>, <Форма>)

Параметры:

<ЗаписьXML> (обязательный)

Тип: [ЗаписьУзловDOM](ЗаписьУзловDOM.md), [ЗаписьFastInfoset](ЗаписьFastInfoset.md), [ЗаписьXML](ЗаписьXML.md).   
Объект, через который осуществляется запись XML.

<Значение> (обязательный)

Записываемое в поток XML значение. Тип параметра определяется совокупностью типов, для которых определена XML-сериализация.  
При записи объекта метаданных выгружаются все записи объекта метаданных, доступные для пользователя.

<ЛокальноеИмя> (обязательный)

Тип: Строка.   
Локальное имя элемента XML, в который будет записано значение.

<URIПространстваИмен> (обязательный)

Тип: Строка.   
URI пространства имен, к которому принадлежит указанное [ЛокальноеИмя](СвойствоXDTO.ЛокальноеИмя.md).

<НазначениеТипа> (необязательный)

Тип: [НазначениеТипаXML](НазначениеТипаXML.md).   
Определяет необходимость назначения типа элементу XML.  
Значение по умолчанию: [Неявное](НазначениеТипаXML.Неявное.md).

<Форма> (необязательный)

Тип: [ФормаXML](ФормаXML.md).   
Форма записи элемента данных в XML.

Описание варианта метода:

Записывает значения в формате XML с указанием локального имени элемента и пространства имен, к которому принадлежит локальное имя.

Описание:

Производит запись значения в формат XML.

Доступность:

Тонкий клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

Примечание:

Значения не всех типов могут быть записаны в формате XML данным методом.  
В XML могут быть записаны значения следующих типов: Неопределено, Null, Булево, Число, Строка, Дата, Тип, [УникальныйИдентификатор](УникальныйИдентификатор.md), [ДвоичныеДанные](ДвоичныеДанные.md), [ХранилищеЗначения](ХранилищеЗначения.md), [ОписаниеТипов](ОписаниеТипов.md), а также ссылки объектов данных и сами объекты данных, наборы записей регистров, менеджер значения константы. В дальнейшем список может пополняться.  
При попытке записи значения неподходящего типа будет вызвано исключение.

Использование в версии:

Доступен, начиная с версии 8.1.
