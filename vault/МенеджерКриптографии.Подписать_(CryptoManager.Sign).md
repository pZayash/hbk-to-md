# МенеджерКриптографии.Подписать

МенеджерКриптографии (CryptoManager)

Подписать (Sign)

Доступен, начиная с версии 8.2.

Вариант синтаксиса: Данные подписи сохраняются в файл

Синтаксис:

Подписать(<ИсходныеДанные>, <ВыходныеДанные>, <Сертификат>, <ТипПодписи>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Исходные данные для подписывания.  
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).

<ВыходныеДанные> (обязательный)

Тип: [Строка](lang__def_String.md), [Поток](Поток_(Stream).md), [ПотокВПамяти](ПотокВПамяти_(MemoryStream).md), [ФайловыйПоток](ФайловыйПоток_(FileStream).md).   
Имя файла или поток, в который будут выгружены данные подписи.  
Если [Неопределено](lang__def_Undefined.md), то результат работы метода вернется в возвращаемом значении типа [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).

<Сертификат> (необязательный)

Тип: [СертификатКриптографии](СертификатКриптографии_(CryptoCertificate).md).   
Используемый сертификат криптографии.  
Если параметр не указан, то в хранилище персональных сертификатов должен быть ровно один сертификат. В противном случае генерируется исключительная ситуация.

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии_(CryptoSignatureType).md).   
Тип подписи криптографии.  
Методы, отвечающие за установку подписи, поддерживают только следующие типы подписей:

- CMS (CMS) ,
- CAdESBES (CAdESBES) ,
- CAdEST (CAdEST) ,
- CAdESC (CAdESC) ,
- CAdESXLongType2 (CAdESXLongType2) ,
- CAdESAv3 (CAdESAv3) .

  
Значение по умолчанию: [CAdEST](ТипПодписиКриптографии.CAdEST_(CryptoSignatureType.CAdEST).md).

Описание варианта метода:

Данные подписи записываются в файл. Возвращает [Неопределено](lang__def_Undefined.md).

Возвращаемое значение:

Тип: [Неопределено](lang__def_Undefined.md), [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md).   
При сохранении подписи в файл возвращает [Неопределено](lang__def_Undefined.md).

Описание:

Создает подпись в формате CMS (базируется на PKCS#7) и возвращает ее как [ДвоичныеДанные](ДвоичныеДанные_(BinaryData).md) или сохраняет ее в файл.   
Подпись создается алгоритмами, которые заданы свойствами [АлгоритмХеширования](МенеджерКриптографии.АлгоритмХеширования_(CryptoManager.HashAlgorithm).md) и [АлгоритмПодписи](МенеджерКриптографии.АлгоритмПодписи_(CryptoManager.SignAlgorithm).md).  
В данные подписи исходный файл не включается.   
В данные подписи могут быть включены сертификаты, в зависимости от свойства [ВключениеСертификатовВПодпись](МенеджерКриптографии.ВключениеСертификатовВПодпись_(CryptoManager.IncludeCertificatesInSignature).md).  
Если сертификат не указан, то для подписи используется единственный установленный персональный сертификат.  
  
Для работы метода на веб-клиенте необходимо предварительно подключить расширение работы с криптографией. В веб-клиенте метод поддерживается только в браузере Internet Explorer, если в конфигурации используется режим синхронных вызовов расширений платформы и внешних компонент.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

Примечание:

Ошибочной является ситуация, когда персональные сертификаты не установлены.  
Ошибочной является ситуация, когда используемый для подписи сертификат не связан с контейнером ключей.
