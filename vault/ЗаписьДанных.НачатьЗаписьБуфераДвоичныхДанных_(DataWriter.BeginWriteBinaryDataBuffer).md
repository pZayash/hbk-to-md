ЗаписьДанных (DataWriter)

НачатьЗаписьБуфераДвоичныхДанных (BeginWriteBinaryDataBuffer)

Доступен, начиная с версии 8.3.8.

Вариант синтаксиса: Запись буфера

Синтаксис:

НачатьЗаписьБуфераДвоичныхДанных(<ОписаниеОповещения>, <Буфер>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения операции со следующими параметрами:

- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<Буфер> (обязательный)

Тип: [БуферДвоичныхДанных](БуферДвоичныхДанных_(BinaryDataBuffer).md).   
Буфер, который будет записан в поток.

Описание варианта метода:

Записать всё содержимое заданного буфера в целевой поток.

Вариант синтаксиса: Запись части байтов из буфера

Синтаксис:

НачатьЗаписьБуфераДвоичныхДанных(<ОписаниеОповещения>, <Буфер>, <ПозицияВБуфере>, <Количество>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения работы метода со следующими параметрами:

- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<Буфер> (обязательный)

Тип: [БуферДвоичныхДанных](БуферДвоичныхДанных_(BinaryDataBuffer).md).   
Буфер двоичных данных, который используется в качестве источника данных для записи в поток.

<ПозицияВБуфере> (обязательный)

Тип: [Число](lang__def_Number.md).   
Позиция в буфере, начиная с которой выполняется чтение байтов для записи в поток.

<Количество> (обязательный)

Тип: [Число](lang__def_Number.md).   
Количество байтов, которые требуется записать в поток.

Описание варианта метода:

Начинает запись заданного количества байтов из заданного буфера, начиная с заданной позиции.

Описание:

Вызывает начало запись байтов из буфера двоичных данных в целевой поток.

Доступность:

Тонкий клиент, веб-клиент, толстый клиент.
