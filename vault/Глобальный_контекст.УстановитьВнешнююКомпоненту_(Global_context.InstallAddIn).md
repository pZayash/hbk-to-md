# Глобальный контекст.УстановитьВнешнююКомпоненту

Глобальный контекст (Global context)

УстановитьВнешнююКомпоненту (InstallAddIn)

Не рекомендуется использовать, начиная с версии 8.3.16.

Рекомендуется использовать:

- [НачатьУстановкуВнешнейКомпоненты](Глобальный_контекст.НачатьУстановкуВнешнейКомпоненты_(Global_context.BeginInstallAddIn).md)

Доступен, начиная с версии 8.2.

Синтаксис:

УстановитьВнешнююКомпоненту(<Местоположение>)

Параметры:

<Местоположение> (обязательный)

Тип: [Строка](lang__def_String.md).   
Строка, определяющая местоположение внешнего компонента.  
В качестве строки может использоваться:

- полное имя макета, в котором хранится внешний компонент в архиве;
- URL к внешнему компоненту, упакованному в ZIP-архив, в формате, аналогичном [ПолучитьНавигационнуюСсылку](Глобальный_контекст.ПолучитьНавигационнуюСсылку_(Global_context.GetURL).md)().

Описание:

Доставляет объект внешнего компонента с сервера на клиент, после чего он становится доступен для метода [ПодключитьВнешнююКомпоненту](Глобальный_контекст.ПодключитьВнешнююКомпоненту_(Global_context.AttachAddIn).md).

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, толстый клиент, мобильное приложение (клиент).

Примечание:

Метод работает только с компонентами, хранящимися в архиве.  
Если для конфигурации свойство [РежимИспользованияМодальности](ОбъектМетаданныхКонфигурация.РежимИспользованияМодальности_(ConfigurationMetadataObject.ModalityUseMode).md) установлено в [НеИспользовать](РежимИспользованияМодальности.НеИспользовать_(ModalityUseMode.DontUse).md), следует использовать метод [НачатьУстановкуВнешнейКомпоненты](Глобальный_контекст.НачатьУстановкуВнешнейКомпоненты_(Global_context.BeginInstallAddIn).md).
