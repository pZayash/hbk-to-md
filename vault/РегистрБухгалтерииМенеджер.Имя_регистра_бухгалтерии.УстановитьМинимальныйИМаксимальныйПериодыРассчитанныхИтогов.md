РегистрБухгалтерииМенеджер.<Имя регистра бухгалтерии> (AccountingRegisterManager.<Accounting register name>)

УстановитьМинимальныйИМаксимальныйПериодыРассчитанныхИтогов (SetMinAndMaxTotalsPeriods)

Доступен, начиная с версии 8.3.3.

Синтаксис:

УстановитьМинимальныйИМаксимальныйПериодыРассчитанныхИтогов(<МинимальныйПериод>, <МаксимальныйПериод>)

Параметры:

<МинимальныйПериод> (обязательный)

Тип: Дата.   
Начало периода, с которого должны быть рассчитаны итоги.

<МаксимальныйПериод> (обязательный)

Тип: Дата.   
Конец периода, по который должны быть рассчитаны итоги.

Описание:

Устанавливает период, в котором рассчитаны итоги. Влияет на производительность получения остатков и оборотов по данному регистру, а также на производительность регламентных операций типа восстановления базы данных.   
При установке может выполняться расчет итогов за новые открытые периоды.   
Периодичность расчета месяц.

Доступность:

Сервер, толстый клиент, внешнее соединение.

Пример:

|  |
| --- |
| Регистры.Остатки.УстановитьМинимальныйИМакимальныйПериодРассчитанныхИтогов(МинДата, МаксДата); |
