Описание:

Описывает параметры прокси-серверов для различных протоколов. Допустимые протоколы для использования в объекте [ИнтернетПрокси](ИнтернетПрокси_(InternetProxy).md) задаются строками http, https, ftp и ftps.  
Позволяет использовать аутентификацию по имени пользователя и паролю (Basic-аутентификация) или NTLM (Integrated Security)-аутентификацию (в версии 1С:Предприятия под Windows).   
NTLM-аутентификация используется по умолчанию для прокси-серверов, поддерживающих данный способ аутентификации, и не требует дополнительных настроек.  
Для того, чтобы использовать Basic-аутентификацию, следует явно отключить использование NTLM-аутентификации в файле inetcfg.xml или установив в значение [Ложь](lang__def_BooleanFalse.md) параметр <ИспользоватьАутентификациюОС> метода [Установить](ПараметрыМакетаТабличногоДокумента.Установить_(SpreadsheetDocumentTemplateParameters.Set).md).  
В режиме совместимости конфигурации с [Версия8\_3\_6](РежимСовместимости.Версия8_3_6_(CompatibilityMode.Version8_3_6).md), NTLM-аутентификация используется по умолчанию для прокси-серверов, поддерживающих данный способ аутентификации, и не требует дополнительных настроек, т.е. NTLM-аутентификация будет использоваться, если в конструкторе объекта [HTTPСоединение](HTTPСоединение_(HTTPConnection).md) прокси не задан или задан, но без указания имени пользователя и пароля. Задание в конструкторе объекта [HTTPСоединение](HTTPСоединение_(HTTPConnection).md) прокси с указанным именем пользователя и паролем отключает NTLM-аутентификацию, для аутентификации в этом случае используется HTTP Basic-аутентификация.

Доступность:

Тонкий клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.
