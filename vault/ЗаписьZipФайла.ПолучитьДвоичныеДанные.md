ЗаписьZipФайла (ZipFileWriter)

ПолучитьДвоичныеДанные (GetBinaryData)

Не рекомендуется использовать, начиная с версии 8.3.26.

Рекомендуется использовать:

- [ПолучитьДвоичныеДанные](ЗаписьФайлаАрхива.ПолучитьДвоичныеДанные.md)

Доступен, начиная с версии 8.3.10.

Синтаксис:

ПолучитьДвоичныеДанные(<Пароль>, <Комментарий>, <МетодСжатия>, <УровеньСжатия>, <МетодШифрования>)

Параметры:

<Пароль> (необязательный)

Тип: Строка.   
Пароль, который будет назначен архиву.   
Если пароль не назначен или является пустой строкой, то шифрование не происходит.

<Комментарий> (необязательный)

Тип: Строка.   
Комментарий, описывающий ZIP-файл.

<МетодСжатия> (необязательный)

Тип: [МетодСжатияZIP](МетодСжатияZIP.md).   
Метод сжатия, которым будет сжиматься архив.  
Значение по умолчанию: [Сжатие](МетодСжатияZIP.Сжатие.md).

<УровеньСжатия> (необязательный)

Тип: [УровеньСжатияZIP](УровеньСжатияZIP.md).   
Уровень сжатия данных.  
Значение по умолчанию: [Оптимальный](УровеньСжатияZIP.Оптимальный.md).

<МетодШифрования> (необязательный)

Тип: [МетодШифрованияZIP](МетодШифрованияZIP.md).   
Метод шифрования, которым будет зашифрован архив.  
Значение по умолчанию: [Zip20](МетодШифрованияZIP.Zip20.md).

Возвращаемое значение:

Тип: [ДвоичныеДанные](ДвоичныеДанные.md).   

Описание:

Получает ZIP-архив в виде двоичных данных и закрывает файл.  
При использовании данного метода объект [ЗаписьZipФайла](ЗаписьZipФайла.md) должен быть создан с помощью конструктора без параметров.   
При этом открывать архив явно с помощью метода [Открыть](ЗаписьZipФайла.Открыть.md) не требуется.  
При завершении работы данный метод закрывает архив, аналогично методу [Записать](ЗаписьZipФайла.Записать.md).

Доступность:

Тонкий клиент, сервер, толстый клиент, внешнее соединение.

Использование в версии:

Доступен, начиная с версии 8.3.10.

Не рекомендуется использовать, начиная с версии 8.3.26.
