Глобальный контекст (Global context)

ПолучитьФайлыССервераАсинх (GetFilesFromServerAsync)

Доступен, начиная с версии 8.3.15.

Вариант синтаксиса: С диалогом

Синтаксис:

ПолучитьФайлыССервераАсинх(<ПолучаемыеФайлы>, <ПараметрыДиалогаПолученияФайлов>, <ПараметрыПолученияАрхиваФайлов>)

Параметры:

<ПолучаемыеФайлы> (обязательный)

Тип: [Массив](Массив.md).   
Массив объектов типа [ОписаниеПередаваемогоФайла](ОписаниеПередаваемогоФайла.md).  
Каждый объект описывает получаемый файл:

- [Хранение](ОписаниеПередаваемогоФайла.Хранение.md) - расположение данных во временном хранилище или в информационной базе. Значение не должно быть равно пустой строке. В противном случае будет сгенерировано сообщение об ошибке.
- [Имя](ОписаниеПередаваемогоФайла.Имя.md) - либо полный путь к файлу, либо относительный путь к файлу. Относительный путь будет объединен со значением параметра <БазовыйКаталог>.

<ПараметрыДиалогаПолученияФайлов> (необязательный)

Тип: [ПараметрыДиалогаПолученияФайлов](ПараметрыДиалогаПолученияФайлов.md).   
Структура, содержащая параметры диалога, который будет показан пользователю.

<ПараметрыПолученияАрхиваФайлов> (необязательный)

Тип: [ПараметрыПолученияАрхиваФайлов](ПараметрыПолученияАрхиваФайлов.md).   
Параметры получения файлов в виде архива. Если параметр не задан, то получаемые файлы не будут упакованы в архив.

Описание варианта метода:

Начинает получение файлов, отображая диалог, и сохраняет их в локальную файловую систему пользователя.   
  
Результат выполнения обещания заполняется сразу (в веб-клиенте - до показа диалога, в остальных клиентах - после показа диалога) значением Неопределено.  
  
В веб-клиенте расширение работы с файлами не требуется, но используется, если оно уже установлено. В веб-клиенте, если расширение работы с файлами не установлено, настройки из параметра <ПараметрыДиалогаПолученияФайлов> игнорируются.

Вариант синтаксиса: Без диалога

Синтаксис:

ПолучитьФайлыССервераАсинх(<ПолучаемыеФайлы>, <БазовыйКаталог>, <ПараметрыПолученияАрхиваФайлов>)

Параметры:

<ПолучаемыеФайлы> (обязательный)

Тип: [Массив](Массив.md).   
Массив объектов типа [ОписаниеПередаваемогоФайла](ОписаниеПередаваемогоФайла.md).  
Каждый объект описывает получаемый файл:

- [Хранение](ОписаниеПередаваемогоФайла.Хранение.md) - расположение данных во временном хранилище или в информационной базе. Значение не должно быть равно пустой строке. В противном случае будет сгенерировано сообщение об ошибке.
- [Имя](ОписаниеПередаваемогоФайла.Имя.md) - либо полный путь к файлу, либо относительный путь к файлу. Относительный путь будет объединен со значением параметра <БазовыйКаталог>.

<БазовыйКаталог> (необязательный)

Тип: Строка.   
Устанавливает базовый каталог, относительно которого определяется конечное расположение каждого из файлов.  
Если среди описаний получаемых файлов есть относительные пути или установлен режим получения архива [ПолучатьАрхивВсегда](РежимПолученияАрхиваФайлов.ПолучатьАрхивВсегда.md), то параметр <БазовыйКаталог> должен быть задан обязательно.

<ПараметрыПолученияАрхиваФайлов> (необязательный)

Тип: [ПараметрыПолученияАрхиваФайлов](ПараметрыПолученияАрхиваФайлов.md).   
Параметры получения файлов в виде архива. Если параметр не задан, то получаемые файлы не будут упакованы в архив.

Описание варианта метода:

Начинает получение файлов и сохраняет их в локальную файловую систему пользователя.  
В веб-клиенте необходимо расширение работы с файлами.  
  
Результат выполнения обещания - массив объектов типа [ОписаниеПереданногоФайла](ОписаниеПереданногоФайла.md).

Возвращаемое значение:

Тип: [Обещание](Обещание.md).   

Описание:

Асинхронная функция. Получает файлы с сервера.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, толстый клиент, мобильное приложение (клиент).
