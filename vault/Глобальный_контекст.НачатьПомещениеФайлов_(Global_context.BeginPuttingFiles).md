# Глобальный контекст.НачатьПомещениеФайлов

**↑** [Главная](_index.md) › [Объекты](_index__Объекты.md) › [Глобальный контекст](Глобальный_контекст.md) › [Методы](Методы.md) › [Процедуры и функции работы с файлами](Процедуры_и_функции_работы_с_файлами.md)

Глобальный контекст (Global context)

НачатьПомещениеФайлов (BeginPuttingFiles)

Не рекомендуется использовать, начиная с версии 8.3.15.

Рекомендуется использовать:

- [НачатьПомещениеФайлаНаСервер](Глобальный_контекст.НачатьПомещениеФайлаНаСервер_(Global_context.BeginPutFileToServer).md)
- [НачатьПомещениеФайловНаСервер](Глобальный_контекст.НачатьПомещениеФайловНаСервер_(Global_context.BeginPutFilesToServer).md)

Доступен, начиная с версии 8.3.6.

Синтаксис:

НачатьПомещениеФайлов(<ОписаниеОповещенияОЗавершении>, <ПомещаемыеФайлы>, <Интерактивно>, <УникальныйИдентификаторФормы>, <ОписаниеОповещенияПередНачаломПомещенияФайлов>, <ОписаниеОповещенияОХодеВыполнения>)

Параметры:

<ОписаниеОповещенияОЗавершении> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана после завершения со следующими параметрами:

- <ПомещенныеФайлы> – массив объектов типа [ОписаниеПереданногоФайла](ОписаниеПереданногоФайла_(TransferredFileDescription).md) или [Неопределено](lang__def_Undefined.md), если файлы не получены.
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<ПомещаемыеФайлы> (необязательный)

Тип: [Массив](Массив_(Array).md), [ДиалогВыбораФайла](ДиалогВыбораФайла_(FileDialog).md), [Строка](lang__def_String.md).   
Параметр может содержать массив объектов типа [ОписаниеПередаваемогоФайла](ОписаниеПередаваемогоФайла_(TransferableFileDescription).md), или массив объектов типа [СсылкаНаФайл](СсылкаНаФайл_(FileRef).md), или объект типа [ДиалогВыбораФайла](ДиалогВыбораФайла_(FileDialog).md) или [Строка](lang__def_String.md).  
Для массива объектов типа [ОписаниеПередаваемогоФайла](ОписаниеПередаваемогоФайла_(TransferableFileDescription).md)в зависимости от его содержимого изменяется хранилище, в котором файлы будут сохранены:

- Массив объектов может быть заполнен полностью. - В этом случае известен набор помещаемых файлов и адреса, по которым они должны быть сохранены.
- Массив объектов может содержать только имена элементов. - В этом случае, набор помещаемых файлов определен, а места хранения неизвестны. Файлы будут размещены в новых временных хранилищах.

Если параметр задан строкой, то эта строка должна иметь вид строки поиска для метода [НайтиФайлы](Глобальный_контекст.НайтиФайлы_(Global_context.FindFiles).md).  
Параметр <ПомещаемыеФайлы> указывается в зависимости от значения параметра <Интерактивно>:

- Если параметр <Интерактивно> имеет значение [Истина](lang__def_BooleanTrue.md), параметр <ПомещаемыеФайлы> может быть не задан или должен иметь тип [ДиалогВыбораФайла](ДиалогВыбораФайла_(FileDialog).md).
- Если параметр <Интерактивно> имеет значение [Ложь](lang__def_BooleanFalse.md), то параметр <ПомещаемыеФайлы> должен быть задан и иметь тип [Строка](lang__def_String.md) или массив объектов типа [ОписаниеПередаваемогоФайла](ОписаниеПередаваемогоФайла_(TransferableFileDescription).md), или массив объектов типа [СсылкаНаФайл](СсылкаНаФайл_(FileRef).md).

В веб-клиенте, если расширение работы с файлами не установлено, то параметр <ПомещаемыеФайлы> должен иметь тип [ДиалогВыбораФайла](ДиалогВыбораФайла_(FileDialog).md) или массив объектов типа [СсылкаНаФайл](СсылкаНаФайл_(FileRef).md).

<Интерактивно> (необязательный)

Тип: [Булево](lang__Булево_(Boolean).md).   
Указывает использование интерактивного режима, при котором пользователю показывается диалог, в котором он может выбрать набор помещаемых файлов.  
Значение по умолчанию: Истина.

<УникальныйИдентификаторФормы> (необязательный)

Тип: [УникальныйИдентификатор](УникальныйИдентификатор_(UUID).md).   
Уникальный идентификатор формы, который можно получить с помощью свойства [УникальныйИдентификатор](ФормаКлиентскогоПриложения.УникальныйИдентификатор_(ClientApplicationForm.UUID).md) формы клиентского приложения.   
Файл помещается во временное хранилище и автоматически удаляется после удаления объекта формы.  
Если параметр не указан, помещенное значение будет удалено, после очередного запроса сервера из общего модуля, при контекстном и неконтекстном серверном вызове из формы, при серверном вызове из модуля команды или при получении формы.

<ОписаниеОповещенияПередНачаломПомещенияФайлов> (необязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана непосредственно перед началом помещения файлов во временное хранилище. Оповещение будет вызвано со следующими параметрами:

- <ПомещаемыеФайлы> - массив элементов типа [СсылкаНаФайл](СсылкаНаФайл_(FileRef).md), готовые к помещению во временное хранилище.
- <ОтказОтПомещенияВсехФайлов> - признак отказа от дальнейшего помещения всех файлов. Если в теле процедуры-обработчика установить данному параметру значение [Истина](lang__def_BooleanTrue.md), то помещение всех файлов будет отменено, обработчик <ОписаниеОповещенияОЗавершении> не вызовется. Значение по умолчанию: [Ложь](lang__def_BooleanFalse.md). Возвращаемый параметр.
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

Если параметр не указан или имеет тип [Неопределено](lang__def_Undefined.md), то перед помещением файлов никакая процедура вызвана не будет.

<ОписаниеОповещенияОХодеВыполнения> (необязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит описание процедуры, которая будет вызвана несколько раз по мере помещения файлов. Количество раз и время вызова не определено строго. Оповещение будет вызвано со следующими параметрами:

- <ПомещаемыйФайл> - ссылка на файл, помещаемый во временное хранилище.
- <Помещено> - количество уже помещенных во временное хранилище байт (для одного файла).
- <РазмерПомещаемогоФайла> - размер файла в байтах.
- <ОтказОтПомещенияФайла> - признак отказа от дальнейшего помещения файла. Если в теле процедуры-обработчика установить данному параметру значение [Истина](lang__def_BooleanTrue.md), то помещение файла будет отменено. Значение по умолчанию: [Ложь](lang__def_BooleanFalse.md). Возвращаемый параметр.
- <ПомещеноВсего> - количество уже помещенных во временное хранилище байт (для всех файлов).
- <РазмерПомещаемыхФайловВсего> - суммарный размер всех файлов в байтах.
- <ОтказОтПомещенияВсехФайлов> - признак отказа от дальнейшего помещения всех файлов. Если в теле процедуры-обработчика установить данному параметру значение [Истина](lang__def_BooleanTrue.md), то помещение всех файлов будет отменено. Значение по умолчанию: [Ложь](lang__def_BooleanFalse.md). Возвращаемый параметр.
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

Если параметр не указан или имеет тип [Неопределено](lang__def_Undefined.md), то в процессе помещения файлов никакая процедура вызвана не будет.

Описание:

Начинает помещение набора файлов из локальной файловой системы во временное хранилище.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, толстый клиент, мобильное приложение (клиент).

Использование в версии:

Доступен, начиная с версии 8.3.6.

Не рекомендуется использовать, начиная с версии 8.3.15.

Описание изменено в версии 8.3.22.
