СправочникМенеджер.<Имя справочника> (CatalogManager.<Catalog name>)

УстановитьИнициализациюПредопределенныхДанных (SetPredefinedDataInitialization)

Доступен, начиная с версии 8.3.5.

Синтаксис:

УстановитьИнициализациюПредопределенныхДанных(<Инициализированы>, <Область>)

Параметры:

<Инициализированы> (необязательный)

Тип: [Булево](lang__Булево_(Boolean).md).   
Указывает, инициализированы ли предопределенные данные в области или нет.  
Значение по умолчанию: Истина.

<Область> (необязательный)

Тип: [Соответствие](Соответствие_(Map).md).   
Данный параметр позволяет задать область объекта, в которой необходимо изменить признак проинициализированности.  
Область описывается соответствием: разделитель - значение. Могут быть использованы только разделители, разделяющие данные в режиме [НезависимоИСовместно](ИспользованиеРазделяемыхДанныхОбщегоРеквизита.НезависимоИСовместно_(CommonAttributeSeparatedDataUse.IndependentlyAndSimultaneously).md).  
Ключом соответствия может быть объект метаданных или имя разделителя.  
Если разделители, переданные в этом параметре, установлены и используются в сеансе, то они игнорируются.  
Значение по умолчанию: [Неопределено](lang__def_Undefined.md).

Описание:

Устанавливает признак инициализации предопределенных данных области.  
Если значение параметра <Инициализированы> = [Ложь](lang__def_BooleanFalse.md), то предопределенные данные должны фактически отсутствовать.

Доступность:

Сервер, толстый клиент, внешнее соединение, мобильное приложение (сервер), мобильный автономный сервер.

Использование в версии:

Доступен, начиная с версии 8.3.5.
