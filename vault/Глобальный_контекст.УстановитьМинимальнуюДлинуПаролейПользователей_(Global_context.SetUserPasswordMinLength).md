Глобальный контекст (Global context)

УстановитьМинимальнуюДлинуПаролейПользователей (SetUserPasswordMinLength)

Доступен, начиная с версии 8.1.

Синтаксис:

УстановитьМинимальнуюДлинуПаролейПользователей(<Длина>)

Параметры:

<Длина> (необязательный)

Тип: [Число](lang__def_Number.md).   
Минимально допустимое количество символов в паролях пользователей.  
Значение по умолчанию: 5.

Описание:

Устанавливает минимальную длину, которую должны иметь пароли пользователей информационной базы.  
Действует только:

- при изменении пароля существующего пользователя или при создании нового пользователя, если включена [АутентификацияСтандартная](ПользовательИнформационнойБазы.АутентификацияСтандартная_(InfoBaseUser.StandardAuthentication).md) или новый пароль не пустой;
- при включении [АутентификацияСтандартная](ПользовательИнформационнойБазы.АутентификацияСтандартная_(InfoBaseUser.StandardAuthentication).md) у существующего пользователя, если пароль пустой.

Метод доступен только из сеанса, в котором все разделители не используются.  
Если для информационной базы установлен параметр "Проверка сложности паролей пользователей", то пароли должны иметь длину не менее 7 символов, а значение параметра "Минимальная длина паролей пользователей" учитывается, только если оно превышает 7.

Доступность:

Сервер, толстый клиент, внешнее соединение.

Использование в версии:

Доступен, начиная с версии 8.1.

Описание изменено в версии 8.3.22.
