# ПланСчетовМенеджер.<Имя плана счетов>.НайтиПоРеквизиту

ПланСчетовМенеджер.<Имя плана счетов> (ChartOfAccountsManager.<Chart of accounts name>)

НайтиПоРеквизиту (FindByAttribute)

Доступен, начиная с версии 8.0.

Синтаксис:

НайтиПоРеквизиту(<ИмяРеквизита>, <ЗначениеРеквизита>, <Родитель>)

Параметры:

<ИмяРеквизита> (обязательный)

Тип: [Строка](lang__def_String.md).   
Имя реквизита, как он задан в конфигураторе, по значению которого осуществляется поиск. Тип значения произвольный, кроме [ХранилищеЗначения](ХранилищеЗначения_(ValueStorage).md) и строк произвольной длины.

<ЗначениеРеквизита> (обязательный)

Тип: Произвольный.   
Значение реквизита, по которому должен выполняться поиск.

<Родитель> (необязательный)

Тип: [ПланСчетовСсылка.\<Имя плана счетов\>](ПланСчетовСсылка.Имя_плана_счетов_(ChartOfAccountsRef.Chart_of_accounts_name).md).   
Родитель, в пределах которого нужно выполнять поиск. Если не указан, то во всех счетах.

Возвращаемое значение:

Тип: [ПланСчетовСсылка.\<Имя плана счетов\>](ПланСчетовСсылка.Имя_плана_счетов_(ChartOfAccountsRef.Chart_of_accounts_name).md).   
Ссылка на найденный элемент счета. Если не существует ни одного элемента с требуемым значением реквизита, то будет возвращена пустая ссылка.

Описание:

Осуществляет поиск элемента по значению реквизита.

Доступность:

Сервер, толстый клиент, внешнее соединение.

Примечание:

Если существует несколько элементов с указанным значением реквизита, то будет найден только один из них.  
Для реквизитов типа [Строка](lang__def_String.md) поиск осуществляется по точному соответствию.

Использование в версии:

Доступен, начиная с версии 8.0.
