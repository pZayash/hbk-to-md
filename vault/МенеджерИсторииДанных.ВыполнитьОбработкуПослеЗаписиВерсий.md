МенеджерИсторииДанных (DataHistoryManager)

ВыполнитьОбработкуПослеЗаписиВерсий (ExecuteAfterWriteVersionsProcessing)

Доступен, начиная с версии 8.3.15.

Вариант синтаксиса: Выполнение обработки после записи версий указанных объектов метаданных

Синтаксис:

ВыполнитьОбработкуПослеЗаписиВерсий(<ВключитьМетаданные>, <ИсключитьМетаданные>)

Параметры:

<ВключитьМетаданные> (необязательный)

Тип: Неопределено, [Массив](Массив.md).   
Объекты метаданных, по данным которых выполняется обработка записей версий.   
Если значение параметра Неопределено или пустой массив, то обработка выполняется по всем объектам метаданных, за исключением указанных в параметре <ИсключитьМетаданные>.  
Значение по умолчанию: Неопределено.

<ИсключитьМетаданные> (необязательный)

Тип: Неопределено, [Массив](Массив.md).   
Объекты метаданных, по данным которых не выполняется обработка записей версий.  
Значение по умолчанию: Неопределено.

Описание варианта метода:

Запускает выполнение обработчика события ОбработкаПослеЗаписиВерсийИсторииДанных для версий, при записи которых был установлен флаг ВыполнятьОбработкуПослеЗаписиВерсии:

- Если метод вызывается из сеанса, в котором разделитель не используется:
  - Для каждой области данных, имеющей версии, требующие обработки, последовательно запускается фоновое задание, в котором выполняется обработка версий данных области.
  - При выполнении обработки устанавливаются все разделители, даже если они не используются в объекте метаданных, с целью вызвать обработчик в сеансе, аналогичном сеансу, в котором выполнялась запись данных.
  - Обработка выполняется в безопасном режиме разделения данных по всем разделителям.
  - Требуется право "Администрирование".
  - Если при выполнении обработки в области генерируется исключение, то последующая обработка в области прерывается, обработка других областей не прерывается.
- Если метод вызывается из сеанса, в котором разделитель используется:
  - Выполняется обработка версий данных текущей области.
  - Требуется право "Администрирование данных".
  - Если при выполнении обработчика генерируется исключение, то последующая обработка прерывается.
  - При определении области данных учитываются все разделители конфигурации.

Вариант синтаксиса: Выполнение обработки после записи версий указанных данных

Синтаксис:

ВыполнитьОбработкуПослеЗаписиВерсий(<Данные>)

Данные, по которым выполняется обновление истории.

Описание варианта метода:

Запускает выполнение обработчика ОбработкаПослеЗаписиВерсийИсторииДанных для версий, при записи которых был установлен флаг ВыполнятьОбработкуПослеЗаписиВерсии.  
Если обработчик не реализован, то записи удаляются из очереди без обработки.  
Обработчик выполняется в текущей области данных.

Описание:

Для версий, при записи которых было установлено свойство ВыполнятьОбработкуПослеЗаписиВерсии, запускает выполнение обработчика события ОбработкаПослеЗаписиВерсийИсторииДанных.

Доступность:

Сервер, толстый клиент, внешнее соединение.

Использование в версии:

Доступен, начиная с версии 8.3.15.
