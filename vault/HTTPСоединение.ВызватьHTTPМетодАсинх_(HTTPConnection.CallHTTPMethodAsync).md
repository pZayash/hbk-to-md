HTTPСоединение (HTTPConnection)

ВызватьHTTPМетодАсинх (CallHTTPMethodAsync)

Доступен, начиная с версии 8.3.21.

Синтаксис:

ВызватьHTTPМетодАсинх(<HTTPМетод>, <HTTPЗапрос>, <ИмяВыходногоФайла>)

Параметры:

<HTTPМетод> (обязательный)

Тип: [Строка](lang__def_String.md).   
Строка, содержащая имя HTTP-метода для запроса.  
Согласно стандарту RFC7230 (https://tools.ietf.org/html/rfc7230), строка должна содержать только символы латинского алфавита и цифры и символы: "!","#", "$", "%", "", "'", "\*", "+", "-", ".", "^", "\_", "`", "|", "~". Использование других символов не допускается.

<HTTPЗапрос> (обязательный)

Тип: [HTTPЗапрос](HTTPЗапрос_(HTTPRequest).md).   
Запрос, передаваемый на сервер.

<ИмяВыходногоФайла> (необязательный)

Тип: [Строка](lang__def_String.md).   
Имя файла, в который записывается тело ответа. Если не задано, то тело ответа может быть получено из объекта [HTTPОтвет](HTTPОтвет_(HTTPResponse).md).  
В веб-клиенте параметр поддерживается только при подключенном расширении работы с файлами.  
Значение по умолчанию: Пустая строка.

Возвращаемое значение:

Тип: [Обещание](Обещание_(Promise).md).   
Результат выполнения обещания – объект типа [HTTPОтвет](HTTPОтвет_(HTTPResponse).md).   
Если задан параметр <ИмяВыходногоФайла>, то тело ответа будет записано в файл. В противном случае, тело ответа можно получить методами [ПолучитьТелоКакСтроку](HTTPСервисОтвет.ПолучитьТелоКакСтроку_(HTTPServiceResponse.GetBodyAsString).md) и [ПолучитьТелоКакДвоичныеДанные](HTTPЗапрос.ПолучитьТелоКакДвоичныеДанные_(HTTPRequest.GetBodyAsBinaryData).md) объекта [HTTPОтвет](HTTPОтвет_(HTTPResponse).md).

Описание:

Асинхронно отправляет данные на указанный адрес для обработки с использованием указанного HTTP-метода.

Доступность:

Тонкий клиент, мобильный клиент, толстый клиент, мобильное приложение (клиент).

Примечание:

При использовании метода GET тело запроса на сервер отправлено не будет.  
При использовании мобильной платформы на устройствах под управлением ОС Android версий 2.3 - 4.4 вызов данного метода с параметром PATCH недоступен и приведет к исключению.  
В веб-клиенте нужно учитывать, что запрос можно отправить только на тот же самый домен, на котором опубликован веб-клиент, или на домен, в котором нет CORS-ограничений на принятие запроса от веб-клиента. В противном случае будет вызвано исключение.  
Если веб-клиент опубликован по HTTPS, то запрос можно отправить, только используя защищенное соединение. В противном случае будет вызвано исключение.

Использование в версии:

Доступен, начиная с версии 8.3.21.
