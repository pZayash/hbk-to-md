МенеджерХранилищаДвоичныхДанных (BinaryDataStorageManager)

УстановитьМинимальныйРазмерЗаписываемыхДанных (SetMinWriteDataSize)

Доступен, начиная с версии 8.3.23.

Синтаксис:

УстановитьМинимальныйРазмерЗаписываемыхДанных(<МинимальныйРазмерЗаписываемыхДанных>)

Параметры:

<МинимальныйРазмерЗаписываемыхДанных> (обязательный)

Тип: Число.   
Минимальный размер записываемых двоичных данных в хранилище (в байтах).

Возвращаемое значение:

Нет

Описание:

Устанавливает минимальный размер двоичных данных (в байтах), которые будут сохраняться в хранилище двоичных данных.   
Все двоичные данные типа [ХранилищеЗначения](ХранилищеЗначения.md), кроме данных из служебных таблиц, размер которых больше или равен указанному, будут попадать в хранилище двоичных данных.  
Установка минимального размера требует монопольного доступа к информационной базе (устанавливается с помощью метода [УстановитьМонопольныйРежим](Глобальный_контекст.УстановитьМонопольныйРежим.md)).  
  
Минимальное значение параметра размера двоичных данных равно 2048 байт.  
При попытке установить меньшее значение, устанавливается значение 2048 байт.  
Вызов метода требует административных прав в сеансе, в котором все разделители не установлены.  
 Если до вызова метода режим монопольного доступа к базе данных не был установлен, то он устанавливается на время вызова. Если установить монопольный доступ невозможно, будет сгенерировано исключение.

Доступность:

Сервер.

Использование в версии:

Доступен, начиная с версии 8.3.23.
