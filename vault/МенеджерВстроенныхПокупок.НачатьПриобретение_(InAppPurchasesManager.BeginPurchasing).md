# МенеджерВстроенныхПокупок.НачатьПриобретение

МенеджерВстроенныхПокупок (InAppPurchasesManager)

НачатьПриобретение (BeginPurchasing)

Доступен, начиная с версии 8.3.18.

Синтаксис:

НачатьПриобретение(<ОписаниеОповещенияОЗавершении>, <Покупка>, <АвтоПодтверждение>)

Параметры:

<ОписаниеОповещенияОЗавершении> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).   
Содержит ссылку на процедуру, которая будет вызвана после завершения процесса покупки. Процедура должна иметь следующие параметры:

- Покупка: тип [ВстроеннаяПокупка](ВстроеннаяПокупка_(InAppPurchase).md)/[Строка](lang__def_String.md) – Покупка, приобретение которой было запрошено. Поле может быть полезным, если одна функция-обработчик используется для нескольких различных покупок.
- Успешно: тип [Булево](lang__Булево_(Boolean).md) – Результат выполнения операции приобретения. [Истина](lang__def_BooleanTrue.md), если покупка успешно приобретена, [Ложь](lang__def_BooleanFalse.md) – если приобретение не было произведено (пользователь отменил покупку в интерфейсе приложения-магазина, отсутствует связь с сервисом-магазином, цифровая подпись квитанции не прошла проверку).
- Квитанция: тип [КвитанцияВстроеннойПокупки](КвитанцияВстроеннойПокупки_(InAppPurchaseReceipt).md) – Квитанция о покупке. В дальнейшем, ее можно дополнительно проверить: самостоятельно, либо с помощью объекта [МенеджерПроверкиВстроенныхПокупок](МенеджерПроверкиВстроенныхПокупок_(InAppPurchasesValidationManager).md) (для этого потребуется передать квитанцию на сервер).
- Дополнительно – Значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения_(CallbackDescription).md).

<Покупка> (обязательный)

Тип: [ВстроеннаяПокупка](ВстроеннаяПокупка_(InAppPurchase).md), [Строка](lang__def_String.md).   
Приобретаемая покупка.

<АвтоПодтверждение> (необязательный)

Тип: [Булево](lang__Булево_(Boolean).md).   
Если параметр установлен в значение [Истина](lang__def_BooleanTrue.md), то будет выполнена локальная проверка квитанции и если проверка успешна, то покупка будет подтверждена автоматически.   
Параметр используется на устройствах под управлением ОС Android. На iOS и Windows Phone игнорируется.   
Проверка квитанции требует наличия ключа проверки встроенных покупок в ресурсах приложения, который указывается в сборщике мобильных приложений в настройках приложения под Android. Если ключ не найден, то генерируется исключение "Ключ проверки квитанции не найден в ресурсах приложения".   
Значение по умолчанию зависит от режима совместимости:

- Версия
- До версии 8.3.18 - [Истина](lang__def_BooleanTrue.md).
- Версия 8.3.18 и последующие – [Ложь](lang__def_BooleanFalse.md).

  
Значение по умолчанию: [Ложь](lang__def_BooleanFalse.md).

Возвращаемое значение:

Тип: [Булево](lang__Булево_(Boolean).md).   

Описание:

Отправляет запрос на оплату покупки. Управление передается интерфейсу проведения платежей, предоставляемому операционной системой. Возврат в приложение мобильной платформы будет произведен в одном из следующих случаев:

- приобретение покупки завершено,
- пользователь отказался от покупки,
- выполнить покупку невозможно (из-за отсутствия связи с сервисом-магазином, из-за отсутствия необходимого количества денежных средств на балансе пользователя).

Метод является асинхронным.  
Данный метод поддерживается не на всех устройствах. Для проверки совместимости устройства перед вызовом данного метода следует использовать метод [ПоддерживаютсяПокупки](МенеджерВстроенныхПокупок.ПоддерживаютсяПокупки_(InAppPurchasesManager.PurchasesSupported).md).

Доступность:

Мобильный клиент, мобильное приложение (клиент).

См. также:

[МенеджерВстроенныхПокупок](МенеджерВстроенныхПокупок_(InAppPurchasesManager).md), метод [ПоддерживаютсяПокупки](МенеджерВстроенныхПокупок.ПоддерживаютсяПокупки_(InAppPurchasesManager.PurchasesSupported).md)  

Использование в версии:

Доступен, начиная с версии 8.3.18.
