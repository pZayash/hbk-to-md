# Кластер серверов.KillProblemProcesses

**↑** [Главная](_index.md) › [Объекты](_index__Объекты.md) › [Средства интеграции и администрирования](Средства_интеграции_и_администрирования.md) › [Менеджер COM-соединений](Менеджер_COM-соединений.md) › [Администрирование кластера серверов](Администрирование_кластера_серверов.md) › [Кластер серверов](Кластер_серверов_(IClusterInfo).md) › [Свойства](Кластер_серверов__Свойства.md)

Кластер серверов (IClusterInfo)

KillProblemProcesses (KillProblemProcesses)

Доступен, начиная с версии 8.3.14.

Использование:

Только чтение.

Описание:

Тип: [Булево](lang__Булево_(Boolean).md).   
Режим принудительного завершения проблемных процессов.   
Если система мониторинга кластера признает менеджер кластера или рабочий процесс проблемным, то в технологическом журнале выводится событие ATTN. Если данное свойство имеет значение [Истина](lang__def_BooleanTrue.md), то проблемный процесс принудительно завершается средствами операционной системы с записью дампа.

Доступность:

Интеграция.

Использование в версии:

Доступен, начиная с версии 8.3.14.
