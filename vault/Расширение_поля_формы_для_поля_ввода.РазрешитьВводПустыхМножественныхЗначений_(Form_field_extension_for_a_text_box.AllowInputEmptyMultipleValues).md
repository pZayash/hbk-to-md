Расширение поля формы для поля ввода (Form field extension for a text box)

РазрешитьВводПустыхМножественныхЗначений (AllowInputEmptyMultipleValues)

Доступен, начиная с версии 8.3.23.

Использование:

Чтение и запись.

Описание:

Тип: [Булево](lang__Булево_(Boolean).md).   
Определяет возможность ввода пустых множественных значений.  
По умолчанию - [Ложь](lang__def_BooleanFalse.md).

- [Истина](lang__def_BooleanTrue.md) – интерактивный ввод пустых множественных значений разрешен:
  - Добавляется пункт контекстного меню "Добавить пустое значение",
  - Свойство [КнопкаОткрытия](Расширение_поля_формы_для_поля_ввода.КнопкаОткрытия_(Form_field_extension_for_a_text_box.OpenButton).md) принимает значение [Истина](lang__def_BooleanTrue.md).
  - Пункт контекстного меню "Добавить пустое значение" отображается недоступным, если пустое значение выбранного типа уже присутствует в списке (в коллекции), либо для выбранного типа отсутствует пустое значение.
- [Ложь](lang__def_BooleanFalse.md) – интерактивный ввод пустых множественных значений запрещен:
  - При добавлении в результате интерактивных действий пустые значения игнорируется. Проверка на пустые значения выполняется после вызова обработчика [ДобавлениеМножественныхЗначений](Расширение_поля_формы_для_поля_ввода.ДобавлениеМножественныхЗначений_(Form_field_extension_for_a_text_box.MultipleValuesAdd).md), перед добавлением значения. Если проверка не проходит, то отображается оповещение пользователю с текстом "Пустое значение не может быть добавлено".

Запрет ввода пустых множественных значений также распространяется на работу стандартного диалога редактирования списка значений, который вызывается при нажатии на кнопку "Открыть" или сочетания клавиш Ctrl+Shift+F4, если в поле ввода содержит данные типа [СписокЗначений](СписокЗначений_(ValueList).md).

Доступность:

Тонкий клиент, веб-клиент, сервер, толстый клиент.

Использование в версии:

Доступен, начиная с версии 8.3.23.
