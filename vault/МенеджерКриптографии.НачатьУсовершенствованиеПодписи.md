МенеджерКриптографии (CryptoManager)

НачатьУсовершенствованиеПодписи (BeginEnhancingSignature)

Доступен, начиная с версии 8.3.20.

Вариант синтаксиса: Данные подписи в возвращаемом значении

Синтаксис:

НачатьУсовершенствованиеПодписи(<ОписаниеОповещения>, <ИсходныеДанные>, <ТипПодписи>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения.md).   
Содержит описание процедуры, которая будет вызвана после завершения усовершенствования со следующими параметрами:

- <ВыходныеДанные> - помещаются данные усовершенствованной подписи типа [ДвоичныеДанные](ДвоичныеДанные.md).
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения.md).

<ИсходныеДанные> (обязательный)

Тип: Строка, [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Исходные данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md) или [ФайловыйПоток](ФайловыйПоток.md).

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии.md).   
Тип подписи криптографии.  
Значение по умолчанию: [CAdEST](ТипПодписиКриптографии.CAdEST.md).

Описание варианта метода:

Данные подписи возвращаются методом как [Обещание](Обещание.md) с результатом [ДвоичныеДанные](ДвоичныеДанные.md).

Вариант синтаксиса: Данные подписи сохраняются в файл

Синтаксис:

НачатьУсовершенствованиеПодписи(<ОписаниеОповещения>, <ИсходныеДанные>, <ВыходныеДанные>, <ТипПодписи>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: [ОписаниеОповещения](ОписаниеОповещения.md).   
Содержит описание процедуры, которая будет вызвана после завершения усовершенствования подписи со следующими параметрами:

- <ИмяФайла> - имя файла, в который осуществлен вывод.
- <ДополнительныеПараметры> - значение, которое было указано при создании объекта [ОписаниеОповещения](ОписаниеОповещения.md).

<ИсходныеДанные> (обязательный)

Тип: Строка, [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Исходные данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как [ДвоичныеДанные](ДвоичныеДанные.md), [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md) или [ФайловыйПоток](ФайловыйПоток.md).

<ВыходныеДанные> (обязательный)

Тип: Строка, [Поток](Поток.md), [ПотокВПамяти](ПотокВПамяти.md), [ФайловыйПоток](ФайловыйПоток.md).   
Имя файла или поток, в который будут выгружены данные подписи.

<ТипПодписи> (необязательный)

Тип: [ТипПодписиКриптографии](ТипПодписиКриптографии.md).   
Тип подписи криптографии.  
Значение по умолчанию: [CAdEST](ТипПодписиКриптографии.CAdEST.md).

Описание варианта метода:

Данные подписи записываются в файл. Возвращает [Обещание](Обещание.md) с результатом Неопределено

Описание:

Инициирует выполнение усовершенствования подписи в формате CAdES-BES.  
При создании усовершенствованной подписи выполняется обращение к серверам меток времени, адреса которых указаны в поле [АдресаСерверовМетокВремени](МенеджерКриптографии.АдресаСерверовМетокВремени.md).  
Поддерживаемые форматы:

- [CAdEST](ТипПодписиКриптографии.CAdEST.md),
- [CAdESC](ТипПодписиКриптографии.CAdESC.md),
- [CAdESXLongType2](ТипПодписиКриптографии.CAdESXLongType2.md),
- [CAdESAv3](ТипПодписиКриптографии.CAdESAv3.md).

По завершении будет вызвана процедура, указанная в <ОписаниеОповещения>.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

Пример:

|  |
| --- |
| МенеджерКриптографии.НачатьУсовершенствованиеПодписи(         Оповещение, ПодписьCAdESBES, Результат, ТипПодписиКриптографии.CAdESXLongType2); |

Использование в версии:

Доступен, начиная с версии 8.3.20.

Описание изменено в версии 8.3.24.
