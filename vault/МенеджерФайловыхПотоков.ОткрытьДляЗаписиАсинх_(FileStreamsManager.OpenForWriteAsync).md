# МенеджерФайловыхПотоков.ОткрытьДляЗаписиАсинх

МенеджерФайловыхПотоков (FileStreamsManager)

ОткрытьДляЗаписиАсинх (OpenForWriteAsync)

Доступен, начиная с версии 8.3.18.

Синтаксис:

ОткрытьДляЗаписиАсинх(<ИмяФайла>, <РазмерБуфера>)

Параметры:

<ИмяФайла> (обязательный)

Тип: [Строка](lang__def_String.md).   
Имя открываемого файла.

<РазмерБуфера> (необязательный)

Тип: [Число](lang__def_Number.md).   
Размер буфера для операций с файлом (в байтах).  
При записи в поток данные сначала пишутся в буфер, а при заполнении буфера - данные перемещаются на диск.  
Целое число больше 0, максимальный размер 2^32-1.

Возвращаемое значение:

Тип: [Обещание](Обещание_(Promise).md).   
Результат выполнения обещания - объект типа [ФайловыйПоток](ФайловыйПоток_(FileStream).md) для файла с указанным именем.

Описание:

Асинхронная функция. Открывает существующий файл для записи. Если файл не существует, то создаётся новый файл. Запись в существующий файл производится с начала файла поверх существующих данных. Файл открывается в режиме разделяемого чтения.

Доступность:

Тонкий клиент, толстый клиент.
